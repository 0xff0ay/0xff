## Overview

SMTP (Simple Mail Transfer Protocol) is the standard protocol for sending email across the Internet. It operates on multiple ports with different security contexts. **Port 465** was originally designated for SMTPS (implicit TLS), deprecated, then re-assigned by IANA in 2018 for message submission over TLS. SMTP is a critical penetration testing target because it enables **user enumeration**, **open relay abuse**, **email spoofing**, **credential harvesting**, **phishing delivery**, **NTLM hash capture**, and frequently exposes **internal infrastructure details** through headers and bounce messages.

::card-group
  ::card
  ---
  title: Default Ports
  icon: i-lucide-network
  ---
  `25/TCP` (SMTP relay/MTA-to-MTA), `465/TCP` (SMTPS — implicit TLS submission), `587/TCP` (MSA — message submission with STARTTLS), `2525/TCP` (alternate SMTP)
  ::

  ::card
  ---
  title: Protocol
  icon: i-lucide-cable
  ---
  Text-based command/response protocol over TCP. Commands include `HELO/EHLO`, `MAIL FROM`, `RCPT TO`, `DATA`, `AUTH`, `VRFY`, `EXPN`, `STARTTLS`. Responses use 3-digit status codes.
  ::

  ::card
  ---
  title: Authentication
  icon: i-lucide-lock
  ---
  `AUTH PLAIN`, `AUTH LOGIN`, `AUTH CRAM-MD5`, `AUTH DIGEST-MD5`, `AUTH NTLM`, `AUTH GSSAPI` (Kerberos), `AUTH XOAUTH2`. Port 25 often unauthenticated (relay). Ports 465/587 typically require authentication for submission.
  ::

## SMTP Fundamentals

### Port Differences

::collapsible

| Port | Name | Encryption | Purpose | Auth Required |
| ---- | ---- | ---------- | ------- | ------------- |
| `25` | SMTP | None / STARTTLS | MTA-to-MTA relay, legacy submission | Usually no (relay) |
| `465` | SMTPS | Implicit TLS | Client submission over TLS | Yes (submission) |
| `587` | MSA | STARTTLS | Client submission (modern standard) | Yes (submission) |
| `2525` | Alt SMTP | Varies | Alternative when 25 is blocked by ISP | Varies |

::

### SMTP Commands Reference

::code-collapse

```text [smtp-commands.txt]
=== CONNECTION & IDENTIFICATION ===
HELO <hostname>           Identify client (SMTP)
EHLO <hostname>           Identify client (Extended SMTP) - lists capabilities
STARTTLS                  Upgrade to TLS encryption

=== AUTHENTICATION ===
AUTH LOGIN                Base64-encoded username then password
AUTH PLAIN                Base64(\0username\0password) in one step
AUTH CRAM-MD5             Challenge-response (MD5)
AUTH NTLM                 NTLM authentication (Exchange)
AUTH XOAUTH2              OAuth2 token authentication

=== MAIL TRANSFER ===
MAIL FROM:<sender>        Set envelope sender
RCPT TO:<recipient>       Set envelope recipient
DATA                      Begin message body (end with . on own line)
RSET                      Reset current mail transaction

=== USER ENUMERATION ===
VRFY <user>               Verify if user exists (often disabled)
EXPN <list>               Expand mailing list members
RCPT TO:<user>            Also used for user enumeration (response codes)

=== INFORMATION ===
HELP                      List available commands
NOOP                      No operation (keep-alive)
QUIT                      Close connection

=== RESPONSE CODES ===
220    Service ready (banner)
221    Service closing
250    Requested action completed (OK)
251    User not local, will forward
252    Cannot VRFY, but will accept
354    Start mail input
421    Service not available
450    Mailbox unavailable (busy)
451    Local error in processing
452    Insufficient storage
500    Command not recognized
501    Syntax error in parameters
502    Command not implemented
503    Bad sequence of commands
504    Parameter not implemented
530    Authentication required
535    Authentication failed
550    Mailbox unavailable / user unknown
551    User not local
552    Storage allocation exceeded
553    Mailbox name not allowed
554    Transaction failed
```

## User Enumeration

::warning
SMTP user enumeration is one of the **most valuable** techniques during a penetration test. It builds a valid user list for password spraying across multiple services (SMB, RDP, VPN, web apps, etc.).
::

### VRFY Method

```bash [Terminal]
# VRFY directly verifies if a user exists
# 250 = user exists, 550 = user doesn't exist
# Often disabled on modern servers but worth trying

nc <target> 25
# EHLO test
# VRFY admin
# 250 2.1.5 <admin@corp.local>    <-- User exists
# VRFY nonexistent
# 550 5.1.1 User unknown           <-- User doesn't exist

# Quick VRFY test
echo -e "EHLO test\r\nVRFY admin\r\nVRFY root\r\nVRFY postmaster\r\nQUIT\r\n" | nc -w5 <target> 25

# Batch VRFY enumeration
while read user; do
  result=$(echo -e "EHLO test\r\nVRFY $user\r\nQUIT\r\n" | nc -w3 <target> 25 2>/dev/null)
  if echo "$result" | grep -q "^250\|^252"; then
    echo "[+] Valid: $user"
  fi
done < users.txt
```

### EXPN Method

```bash [Terminal]
# EXPN expands mailing list members
# Less commonly enabled than VRFY
# Can reveal multiple users from a single list

nc <target> 25
# EHLO test
# EXPN admin
# EXPN all
# EXPN staff
# EXPN postmaster

# Quick EXPN test
echo -e "EHLO test\r\nEXPN admin\r\nEXPN all\r\nEXPN staff\r\nQUIT\r\n" | nc -w5 <target> 25
```

### RCPT TO Method

::tip
`RCPT TO` enumeration is the **most reliable method** because it's rarely disabled — the server must accept or reject recipients to function. Different response codes indicate valid vs. invalid users.
::

::tabs
  :::tabs-item{icon="i-lucide-terminal" label="Manual RCPT TO"}
  ```bash [Terminal]
  nc <target> 25
  # EHLO test.local
  # MAIL FROM:<test@test.com>
  # RCPT TO:<admin@corp.local>
  # 250 2.1.5 Recipient OK          <-- User exists
  # RCPT TO:<nonexistent@corp.local>
  # 550 5.1.1 User unknown           <-- User doesn't exist
  # RCPT TO:<maybe@corp.local>
  # 252 2.1.5 Cannot verify          <-- User may exist (ambiguous)
  # QUIT

  # Batch RCPT TO enumeration
  while read user; do
    result=$(echo -e "EHLO test\r\nMAIL FROM:<test@test.com>\r\nRCPT TO:<${user}@corp.local>\r\nQUIT\r\n" | nc -w3 <target> 25 2>/dev/null)
    if echo "$result" | grep -q "^250.*Recipient\|^250.*OK"; then
      echo "[+] Valid: $user"
    fi
  done < users.txt
  ```

  :::tabs-item{icon="i-lucide-code" label="Python Script"}
  ```python [smtp_enum.py]
  #!/usr/bin/env python3
  import socket
  import sys

  target = sys.argv[1] if len(sys.argv) > 1 else "<target>"
  port = int(sys.argv[2]) if len(sys.argv) > 2 else 25
  domain = sys.argv[3] if len(sys.argv) > 3 else "corp.local"
  wordlist = sys.argv[4] if len(sys.argv) > 4 else "users.txt"

  with open(wordlist) as f:
      users = [line.strip() for line in f if line.strip()]

  print(f"[*] Enumerating {len(users)} users on {target}:{port} ({domain})")

  s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
  s.settimeout(10)
  s.connect((target, port))
  banner = s.recv(1024).decode()
  print(f"[*] Banner: {banner.strip()}")

  s.send(b'EHLO pentest.local\r\n')
  s.recv(4096)

  s.send(f'MAIL FROM:<test@test.com>\r\n'.encode())
  s.recv(1024)

  valid_users = []
  for user in users:
      email = f"{user}@{domain}"
      s.send(f'RCPT TO:<{email}>\r\n'.encode())
      try:
          resp = s.recv(1024).decode().strip()
          code = resp[:3]
          if code in ['250', '251', '252']:
              print(f"[+] {email} — {resp}")
              valid_users.append(user)
          elif code == '550':
              pass  # User doesn't exist
          else:
              print(f"[?] {email} — {resp}")
      except socket.timeout:
          print(f"[!] Timeout on {email}")
          # Reconnect
          s.close()
          s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
          s.settimeout(10)
          s.connect((target, port))
          s.recv(1024)
          s.send(b'EHLO pentest.local\r\n')
          s.recv(4096)
          s.send(f'MAIL FROM:<test@test.com>\r\n'.encode())
          s.recv(1024)

  s.send(b'QUIT\r\n')
  s.close()

  print(f"\n[*] Found {len(valid_users)} valid users")
  with open("valid_users.txt", "w") as f:
      f.write("\n".join(valid_users))
  print("[*] Saved to valid_users.txt")
  ```

## Pentest Checklist

::steps{level="4"}

#### Banner Grab and Service Identification

Connect to ports 25, 465, and 587. Capture banners and EHLO responses to identify the MTA software (Exchange, Postfix, Exim, Sendmail, OpenSMTPD) and supported capabilities.

#### Extract NTLM Information (Exchange)

Use `smtp-ntlm-info` to obtain domain name, hostname, FQDN, and OS version without authentication from Exchange servers.

#### Enumerate Supported Authentication Mechanisms

Check EHLO response for AUTH methods (LOGIN, PLAIN, CRAM-MD5, NTLM, XOAUTH2). Note if AUTH is offered before STARTTLS (credential interception risk).

#### Test for User Enumeration

Test VRFY, EXPN, and RCPT TO methods to enumerate valid email addresses. RCPT TO is most reliable as it's rarely disabled.

#### Check STARTTLS Enforcement

Verify whether STARTTLS is mandatory before AUTH is offered. If optional, credentials can be sniffed via MITM.

#### Analyze SSL/TLS Configuration

On port 465 and via STARTTLS, check certificate details (hostnames), supported cipher suites, and vulnerable TLS versions.

#### Test for Open Relay

Send test emails with external-to-external, external-to-internal, and internal-to-external sender/recipient combinations. Open relays enable spoofing and phishing.

#### Check SPF, DKIM, and DMARC Records

Query DNS for email authentication records. Missing or weak records (SPF ~all, DMARC p=none) enable email spoofing.

#### Brute Force / Password Spray

Use Hydra, Metasploit, or custom scripts to test credentials on port 587/465. Respect lockout policies.

#### Test Email Spoofing

Send test spoofed emails to verify if the server accepts and delivers spoofed internal-to-internal emails without authentication.

#### Test SMTP Header Injection

If web forms send email, test for CRLF injection in To, From, Subject, and Name fields to inject additional headers or recipients.

#### Test SMTP Smuggling

Check for end-of-data processing differences that allow smuggling additional email messages past security filters.

#### Check for Known Vulnerabilities

Test for Exim RCE (CVE-2019-10149), OpenSMTPD RCE (CVE-2020-7247), Postfix SMTP smuggling, and other version-specific CVEs.

#### Document All Findings

Record evidence of open relay, user enumeration, weak TLS, missing SPF/DMARC, credential exposure, and provide remediation recommendations.

::
