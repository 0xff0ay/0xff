## Overview

IMAP (Internet Message Access Protocol) is a standard protocol for retrieving and managing email messages on a remote mail server. Unlike POP3, IMAP keeps messages on the server and supports folder management, message flags, server-side search, and simultaneous multi-client access. It listens on **TCP port 143** (plaintext/STARTTLS) and **TCP port 993** (implicit TLS/SSL). IMAP is a high-value target during penetration tests because it stores **complete mailbox contents** that often contain credentials, sensitive documents, internal communications, and password reset links.

::card-group
  ::card
  ---
  title: Default Ports
  icon: i-lucide-network
  ---
  `143` (IMAP plaintext / STARTTLS), `993` (IMAPS implicit TLS). Some deployments use non-standard ports like `1143`, `8143`, or `10143`.
  ::

  ::card
  ---
  title: Protocol
  icon: i-lucide-cable
  ---
  Text-based command/response protocol over TCP. Commands are tagged (e.g., `a1 LOGIN user pass`). Supports IDLE, CONDSTORE, QRESYNC, NAMESPACE extensions.
  ::

  ::card
  ---
  title: Authentication
  icon: i-lucide-lock
  ---
  `PLAIN`, `LOGIN`, `CRAM-MD5`, `DIGEST-MD5`, `SCRAM-SHA-1`, `SCRAM-SHA-256`, `NTLM`, `GSSAPI` (Kerberos), `XOAUTH2`. Credentials often transmitted in cleartext on port 143 without STARTTLS.
  ::

  ::card
  ---
  title: Tags
  icon: i-lucide-tags
  ---
  `network pentesting` · `imap` · `email` · `credential harvesting` · `brute force` · `ntlm relay` · `mail exfiltration` · `starttls` · `information disclosure`
  ::

  ```python [imap_enum.py]
  import imaplib
  import ssl

  target = "<target>"
  port = 143       # or 993 for IMAPS
  user = "admin"
  password = "password"

  try:
      # Plaintext IMAP
      imap = imaplib.IMAP4(target, port)

      # Or IMAPS
      # ctx = ssl.create_default_context()
      # ctx.check_hostname = False
      # ctx.verify_mode = ssl.CERT_NONE
      # imap = imaplib.IMAP4_SSL(target, 993, ssl_context=ctx)

      # Or STARTTLS
      # imap = imaplib.IMAP4(target, 143)
      # imap.starttls()

      # Banner / capabilities
      print(f"[*] Banner: {imap.welcome.decode()}")
      typ, caps = imap.capability()
      print(f"[*] Capabilities: {caps[0].decode()}")

      # Login
      imap.login(user, password)
      print(f"[+] Login successful: {user}")

      # List all folders
      typ, folders = imap.list()
      print("\n[*] Mailbox Folders:")
      for folder in folders:
          print(f"    {folder.decode()}")

      # Namespace
      try:
          typ, ns = imap.namespace()
          print(f"\n[*] Namespace: {ns}")
      except Exception:
          pass

      # Select INBOX
      typ, count = imap.select('INBOX')
      print(f"\n[*] INBOX message count: {count[0].decode()}")

      # Search all messages
      typ, msg_ids = imap.search(None, 'ALL')
      ids = msg_ids[0].split()
      print(f"[*] Total messages: {len(ids)}")

      # Search for sensitive content
      searches = [
          ('SUBJECT "password"', 'Password in subject'),
          ('SUBJECT "reset"', 'Password reset'),
          ('SUBJECT "credential"', 'Credentials'),
          ('SUBJECT "vpn"', 'VPN info'),
          ('SUBJECT "ssh"', 'SSH info'),
          ('SUBJECT "key"', 'Key/token info'),
          ('BODY "password"', 'Password in body'),
          ('FROM "admin"', 'From admin'),
          ('FROM "root"', 'From root'),
          ('FROM "noreply"', 'Automated emails'),
      ]

      print("\n[*] Sensitive email search:")
      for query, desc in searches:
          try:
              typ, results = imap.search(None, query)
              hits = results[0].split()
              if hits:
                  print(f"    [+] {desc}: {len(hits)} messages")
          except Exception:
              pass

      # Fetch first 5 message subjects
      print("\n[*] Recent message subjects:")
      for msg_id in ids[-5:]:
          typ, data = imap.fetch(msg_id, '(BODY[HEADER.FIELDS (FROM SUBJECT DATE)])')
          if data[0]:
              print(f"    {data[0][1].decode(errors='ignore').strip()}")

      # Check quota
      try:
          typ, quota = imap.getquotaroot('INBOX')
          print(f"\n[*] Quota: {quota}")
      except Exception:
          pass

      imap.logout()

  except imaplib.IMAP4.error as e:
      print(f"[-] IMAP error: {e}")
  except Exception as e:
      print(f"[-] Connection failed: {e}")
  ```

### NTLM Information Extraction

::tip
Microsoft Exchange IMAP often supports NTLM authentication, which leaks domain information, hostname, and OS version **without credentials**.
::

```bash [Terminal]
# Nmap NTLM info extraction (no auth needed)
nmap -p 143 --script imap-ntlm-info <target>
nmap -p 993 --script imap-ntlm-info <target>

# Manual NTLM probe
# Connect and send AUTHENTICATE NTLM
# The server responds with an NTLM challenge containing:
# - NetBIOS domain name
# - NetBIOS computer name
# - DNS domain name
# - DNS computer name (FQDN)
# - DNS tree name (forest)
# - OS version

# Python NTLM extraction
python3 -c "
import socket, base64

s = socket.socket()
s.connect(('<target>', 143))
banner = s.recv(1024)
print(f'Banner: {banner.decode().strip()}')

# NTLM Type 1 message (negotiate)
ntlm_negotiate = base64.b64encode(
    b'\x4e\x54\x4c\x4d\x53\x53\x50\x00'  # NTLMSSP
    b'\x01\x00\x00\x00'                      # Type 1
    b'\x97\x82\x08\xe2'                      # Flags
    b'\x00\x00\x00\x00\x00\x00\x00\x00'    # Domain
    b'\x00\x00\x00\x00\x00\x00\x00\x00'    # Workstation
).decode()

s.send(f'a1 AUTHENTICATE NTLM\r\n'.encode())
resp = s.recv(1024)
print(f'Challenge prompt: {resp.decode().strip()}')

s.send(f'{ntlm_negotiate}\r\n'.encode())
challenge = s.recv(4096)
print(f'NTLM Challenge (decode for domain info): {challenge.decode().strip()}')

s.send(b'*\r\n')  # Cancel authentication
s.close()
"
```

### Username Enumeration

::note
Some IMAP servers respond differently to valid vs. invalid usernames, enabling username enumeration before password attacks.
::

```bash [Terminal]
# Timing-based enumeration
# Valid users may take longer to respond than invalid users

# SMTP VRFY/EXPN for user enumeration (if SMTP is on same server)
# Then use discovered users against IMAP

smtp-user-enum -M RCPT -U users.txt -t <target>
smtp-user-enum -M EXPN -U users.txt -t <target>

# Finger protocol (if available)
finger @<target>

# LDAP enumeration for email addresses (Active Directory)
```

  ```bash [Terminal]
  # Automated IMAP enumeration
  TARGET=$1
  USER=${2:-""}
  PASS=${3:-""}

  echo "=== IMAP Enumeration: $TARGET ==="

  echo -e "a1 CAPABILITY\r\na2 LOGOUT\r\n" | nc -w5 $TARGET 143 2>/dev/null

  echo -e "\n[*] SSL Certificate (port 993):"
  echo | openssl s_client -connect $TARGET:993 2>/dev/null | \
    openssl x509 -noout -subject -ext subjectAltName 2>/dev/null

  echo -e "\n[*] STARTTLS Status:"
  caps=$(echo -e "a1 CAPABILITY\r\na2 LOGOUT\r\n" | nc -w3 $TARGET 143 2>/dev/null)
  echo "$caps" | grep -i "STARTTLS" && echo "    STARTTLS: Available" || echo "    STARTTLS: Not available"
  echo "$caps" | grep -i "LOGINDISABLED" && echo "    Plaintext LOGIN: Disabled (secure)" || echo "    Plaintext LOGIN: Allowed (insecure!)"

  echo -e "\n[*] Auth Mechanisms:"
  echo "$caps" | grep -oP 'AUTH=[A-Z0-9_-]+' | sort

  echo -e "a1 ID NIL\r\na2 LOGOUT\r\n" | nc -w3 $TARGET 143 2>/dev/null | grep "^\*"

  if [ -n "$USER" ] && [ -n "$PASS" ]; then
    echo -e "\n[*] Authenticated Enumeration:"

    result=$(echo -e "a1 LOGIN $USER $PASS\r\na2 NAMESPACE\r\na3 LIST \"\" \"*\"\r\na4 STATUS INBOX (MESSAGES UNSEEN)\r\na5 GETQUOTAROOT INBOX\r\na6 LOGOUT\r\n" | nc -w5 $TARGET 143 2>/dev/null)
  ```

## Pentest Checklist

::steps{level="4"}

#### Banner Grab and Service Identification

Connect to port 143/993, capture the banner, and identify the IMAP server software (Dovecot, Exchange, Cyrus, Courier, Zimbra, etc.) via banner and ID command.

#### Enumerate Capabilities and Auth Mechanisms

Send `CAPABILITY` command to identify supported authentication mechanisms, STARTTLS availability, and server extensions. Note if `LOGINDISABLED` is absent (plaintext login allowed).

#### Extract NTLM Information (Exchange)

Use `imap-ntlm-info` to obtain domain name, hostname, FQDN, and OS version without authentication from Exchange servers.

#### Check STARTTLS Enforcement

Verify whether the server enforces STARTTLS before allowing LOGIN. If not enforced, credentials can be sniffed in transit.

#### Analyze SSL/TLS Configuration

On port 993, check certificate details (SANs reveal hostnames), supported cipher suites, and vulnerable SSL/TLS versions (SSLv3, TLS 1.0).

#### Test Default and Common Credentials

Try default passwords for the identified mail server software. Test `admin`, `root`, `postmaster`, and service-specific accounts.

#### Brute Force / Password Spray

Use Hydra, Nmap, or Metasploit to brute force or password spray using discovered usernames. Try both `username` and `user@domain.com` formats.

#### Enumerate Usernames via SMTP

If SMTP is available on the same host, use VRFY/RCPT TO enumeration to build a user list for IMAP attacks.

#### Access and Enumerate Mailboxes

After authentication, list all folders (LIST, LSUB), check NAMESPACE for shared folders, and enumerate mailbox sizes.

#### Search for Sensitive Email Content

Search SUBJECT and BODY for passwords, credentials, reset links, VPN configs, SSH keys, API tokens, and financial data.

#### Exfiltrate Complete Mailbox

Dump all emails and attachments using Python scripts, fetchmail, mbsync, or offlineimap for offline analysis.

#### Extract Credentials from Email Content

Parse dumped emails for plaintext passwords, password reset links, API keys, private keys, connection strings, and internal hostnames/IPs.

#### Test Credential Reuse

Try discovered credentials against other services: SMB, RDP, SSH, VPN, web applications, and other mail protocols.

#### Check for Known Vulnerabilities

Test for CVEs specific to the identified IMAP server (Dovecot STARTTLS injection, Cyrus buffer overflows, Zimbra SSRF, Exchange ProxyLogon/ProxyShell).

#### Document All Findings

Record evidence of plaintext authentication, weak passwords, sensitive email content exposure, STARTTLS bypass, and provide remediation recommendations.
