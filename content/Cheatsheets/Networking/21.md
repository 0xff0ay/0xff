## Overview

FTP (File Transfer Protocol) runs on **port 21** by default. It is used for file transfers between a client and server. FTP is inherently insecure — credentials are sent in **cleartext**, making it a high-value target during penetration testing.

> FTP is one of the most commonly misconfigured services. Always check for anonymous access, writable directories, and version-specific exploits.

| Detail       | Value                          |
| ------------ | ------------------------------ |
| **Port**     | `21` (Control), `20` (Data)    |
| **Protocol** | `TCP`                          |
| **Service**  | File Transfer Protocol         |
| **RFC**      | RFC 959                        |
| **Modes**    | Active / Passive               |

## Nmap Scanning

Use Nmap for service detection, script enumeration, and vulnerability scanning against FTP.

### Basic Service Detection

::code-preview
---
class: "[&>div]:*:my-0"
---

Identify the FTP service, version, and OS.

#code
```bash
nmap -sV -sC -p 21 <target>
```
::

### All FTP NSE Scripts

::code-preview
---
class: "[&>div]:*:my-0"
---

Run every FTP-related Nmap script for comprehensive enumeration.

#code
```bash
nmap -p 21 --script="ftp-*" <target>
```
::

### Key Individual Scripts

| Script                   | Purpose                                      |
| ------------------------ | -------------------------------------------- |
| `ftp-anon`               | Check for anonymous login                    |
| `ftp-bounce`             | Test for FTP bounce attack                   |
| `ftp-brute`              | Brute-force FTP credentials                  |
| `ftp-libopie`            | Check for OPIE authentication bypass         |
| `ftp-proftpd-backdoor`   | Detect ProFTPD 1.3.3c backdoor               |
| `ftp-vsftpd-backdoor`    | Detect vsftpd 2.3.4 backdoor (**CVE-2011-2523**) |
| `ftp-vuln-cve2010-4221`  | ProFTPD server Telnet IAC buffer overflow     |
| `ftp-syst`               | Retrieve system information via `SYST` command |

## Anonymous Login

Anonymous FTP access is one of the most common misconfigurations. **Always test this first.**

::code-preview
---

### Manual Anonymous Login

#code
```bash
ftp <target>
# Username: anonymous
# Password: (blank / anonymous / any email)
```
::

Common anonymous credentials to try:

| Username      | Password              |
| ------------- | --------------------- |
| `anonymous`   | *(blank)*             |
| `anonymous`   | `anonymous`           |
| `anonymous`   | `anonymous@email.com` |
| `ftp`         | `ftp`                 |
| `ftp`         | *(blank)*             |
| `guest`       | `guest`               |

::tip
If anonymous login succeeds, immediately check for **writable directories** and **sensitive files** like backups, configs, or credentials.
::

## Manual FTP Enumeration

Once connected, use these FTP commands to enumerate the server.

### Essential FTP Commands

| Command         | Description                              |
| --------------- | ---------------------------------------- |
| `ls -la`        | List all files including hidden          |
| `pwd`           | Print working directory                  |
| `cd <dir>`      | Change directory                         |
| `get <file>`    | Download a file                          |
| `mget *`        | Download all files                       |
| `put <file>`    | Upload a file                            |
| `mkdir <dir>`   | Create a directory                       |
| `binary`        | Switch to binary transfer mode           |
| `ascii`         | Switch to ASCII transfer mode            |
| `passive`       | Toggle passive mode                      |
| `stat`          | Server status and config                 |
| `syst`          | Show system type                         |
| `site help`     | List available SITE commands             |
| `quote PASV`    | Request passive mode (raw)               |
| `quit`          | Disconnect                               |

### Recursive Download

Download everything from the FTP server recursively.

#code
```bash
wget -m --no-passive ftp://anonymous:anonymous@<target>
```

### Recursive Download with cURL

#code
```bash
curl ftp://<target>/ --user anonymous:anonymous -o - | grep -oP '[\w.-]+' | while read f; do curl ftp://<target>/"$f" --user anonymous:anonymous -O; done
```

## FTP Bounce Attack

FTP bounce attacks abuse the `PORT` command to scan internal hosts or bypass firewalls through the FTP server.

::code-preview
---
class: "[&>div]:*:my-0"
---

### Nmap FTP Bounce Scan

#code
```bash
nmap -Pn -v -p 80,443,8080 -b anonymous@<ftp_target> <internal_target>
```
::

::code-preview
---
class: "[&>div]:*:my-0"
---

### Nmap Bounce Check Script

#code
```bash
nmap -p 21 --script ftp-bounce --script-args ftp-bounce.checkhost=<internal_host> <target>
```
::

> FTP bounce attacks are described in **RFC 2577** as a known security risk. Most modern FTP servers disable this by default, but legacy systems remain vulnerable.

## Config File Locations

After gaining access, look for FTP configuration files that may contain credentials or reveal further attack surface.

### Linux

| Server    | Config Path                        |
| --------- | ---------------------------------- |
| vsftpd    | `/etc/vsftpd.conf`                |
| vsftpd    | `/etc/vsftpd/vsftpd.conf`         |
| vsftpd    | `/etc/vsftpd.userlist`             |
| ProFTPD   | `/etc/proftpd/proftpd.conf`       |
| Pure-FTPd | `/etc/pure-ftpd/pure-ftpd.conf`   |
| wu-ftpd   | `/etc/ftpaccess`                   |
| General   | `/etc/ftpusers` (denied users)     |
| Logs      | `/var/log/vsftpd.log`              |
| Logs      | `/var/log/xferlog`                 |

### Windows

| Server        | Config Path                                  |
| ------------- | -------------------------------------------- |
| IIS FTP       | `%SystemRoot%\System32\inetsrv\config\`      |
| FileZilla     | `C:\Program Files\FileZilla Server\`         |
| FileZilla     | `FileZilla Server.xml` (creds in base64)     |
| CesarFTP      | Application directory                        |

::tip
FileZilla Server stores credentials in `FileZilla Server.xml` encoded in **Base64** — not encrypted. Always check this file.
::

## Sensitive Files to Look For

When you gain FTP access, prioritize downloading these file types:

| File Pattern                | Why It Matters                        |
| --------------------------- | ------------------------------------- |
| `*.conf`, `*.cfg`, `*.ini`  | Configuration files with credentials  |
| `*.bak`, `*.old`, `*.orig`  | Backup files with sensitive data      |
| `*.txt`, `*.log`            | Potential password or info leakage    |
| `*.sql`, `*.db`, `*.sqlite` | Database dumps                        |
| `*.key`, `*.pem`, `*.crt`   | Private keys and certificates         |
| `.htpasswd`, `.htaccess`    | Web server authentication files       |
| `wp-config.php`             | WordPress database credentials        |
| `web.config`                | IIS configuration (connection strings)|
| `.ssh/`, `authorized_keys`  | SSH keys for lateral movement         |
| `shadow`, `passwd`          | Linux user credential files           |

## Post-Access Checks

After successfully authenticating (anonymous or otherwise), verify these attack paths:

### 1. Writable Directory Check

::code-preview
---
class: "[&>div]:*:my-0"
---

Test if you can upload files (web shell, SSH keys, etc.).

#code
```bash
ftp> put test.txt
# If successful, check if directory maps to a web root
```
::

### 2. Web Root Overlap

If the FTP directory overlaps with a **web server root** (e.g., `/var/www/html`), upload a web shell:

::code-preview
---
class: "[&>div]:*:my-0"
---

Upload a PHP reverse shell

#code
```bash
ftp> binary
ftp> put shell.php
# Then browse to http://<target>/shell.php
```
::

### 3. SUID / Credential Harvesting

::code-preview
---
class: "[&>div]:*:my-0"
---

Search for interesting files recursively

#code
```bash
# After downloading all files via wget
find . -name "*.conf" -o -name "*.txt" -o -name "*.bak" | xargs grep -il "password\|passwd\|credential\|secret"
```

## Quick Enumeration Checklist

Use this checklist to ensure thorough FTP enumeration:

- [ ] Banner grab → identify software and version
- [ ] Nmap service scan with default scripts (`-sV -sC`)
- [ ] Test **anonymous login** (multiple credential combos)
- [ ] Run all `ftp-*` Nmap scripts
- [ ] Check for **known backdoors** (vsftpd 2.3.4, ProFTPD 1.3.3c)
- [ ] Enumerate **all files and directories** (including hidden)
- [ ] Test **write permissions** (upload test file)
- [ ] Check for **FTP bounce** attack viability
- [ ] Brute-force credentials (Hydra / Medusa / Metasploit)
- [ ] Search for **config files** and **credentials**
- [ ] Check if FTP root overlaps with **web root**
- [ ] Look up version in **searchsploit** and CVE databases
- [ ] Sniff traffic for cleartext credentials if on network
- [ ] Check for **FTPS/SFTP** on alternate ports (990, 22)
