## Overview

::card
---
icon: i-lucide-info
title: What is Port 80?
---
**HTTP** (HyperText Transfer Protocol) operates on **TCP port 80** as the foundation of the World Wide Web. It handles client-server communication for web applications, APIs, and services. HTTP transmits data in **cleartext** (unlike HTTPS/443). Port 80 is the **most commonly attacked port** on the internet — hosting web applications with vast attack surfaces spanning injection flaws, misconfigurations, authentication bypasses, and more.
::

::callout
---
icon: i-lucide-skull
color: red
---
Port 80 is the **#1 target** in penetration testing. Web applications expose enormous attack surfaces — every input field, parameter, header, cookie, and API endpoint is a potential entry point for exploitation.
::

### Manual Enumeration Techniques

::steps{level="4"}

#### HTTP Headers Analysis

```bash [Terminal]
# Full header dump
curl -sI http://<TARGET_IP>

# Verbose connection
curl -sv http://<TARGET_IP> 2>&1

# Check specific headers
curl -sI http://<TARGET_IP> | grep -iE \
  "server|x-powered|x-aspnet|x-frame|content-security|x-xss|strict-transport"

# Security headers check
# Look for MISSING headers:
# - Strict-Transport-Security
# - Content-Security-Policy
# - X-Content-Type-Options
# - X-Frame-Options
# - X-XSS-Protection
# - Referrer-Policy
# - Permissions-Policy

# Check all HTTP methods
for method in GET POST PUT DELETE PATCH OPTIONS HEAD TRACE CONNECT; do
  echo -n "$method: "
  curl -s -o /dev/null -w "%{http_code}" -X $method http://<TARGET_IP>/
  echo
done
```

#### Sensitive File Discovery

```bash [Terminal]
# Check common sensitive files
for file in robots.txt sitemap.xml .htaccess .htpasswd \
  web.config crossdomain.xml clientaccesspolicy.xml \
  .git/HEAD .git/config .svn/entries .env .env.bak \
  composer.json package.json wp-config.php.bak \
  config.php.bak database.yml server-status server-info \
  phpinfo.php info.php test.php .DS_Store Thumbs.db \
  backup.zip backup.tar.gz dump.sql debug.log \
  error.log access.log .well-known/security.txt; do
  
  code=$(curl -s -o /dev/null -w "%{http_code}" "http://<TARGET_IP>/$file")
  if [ "$code" != "404" ]; then
    echo "[${code}] /$file"
  fi
done
```

### Detailed Attack Surface

::field-group
  ::field{name="SQL Injection" type="Critical"}
  Inject SQL queries through user input to dump databases, bypass authentication, read/write files, or execute OS commands. Union-based, blind, error-based, time-based, and out-of-band techniques.
  ::

  ::field{name="Cross-Site Scripting (XSS)" type="High"}
  Reflected, stored, and DOM-based XSS. Inject JavaScript to steal cookies, session tokens, keylog users, or redirect to malicious sites.
  ::

  ::field{name="Local File Inclusion (LFI)" type="Critical"}
  Include local files via path traversal. Read `/etc/passwd`, application source code, log files. Can escalate to RCE via log poisoning, PHP wrappers, or `/proc/self/environ`.
  ::

  ::field{name="Remote File Inclusion (RFI)" type="Critical"}
  Include remote files (attacker-controlled) to achieve direct RCE. Requires `allow_url_include=On` in PHP.
  ::

  ::field{name="Command Injection" type="Critical"}
  OS command execution through unsanitized input. Pipe operators `; | && || $()` to chain commands. Direct path to reverse shell.
  ::

  ::field{name="Server-Side Request Forgery (SSRF)" type="High"}
  Force server to make requests to internal resources, cloud metadata endpoints (169.254.169.254), or other internal services.
  ::

  ::field{name="File Upload Vulnerabilities" type="Critical"}
  Upload web shells by bypassing extension filters, MIME type checks, or content validation. PHP, ASP, JSP web shells for RCE.
  ::

  ::field{name="Insecure Direct Object Reference (IDOR)" type="High"}
  Access other users' data by manipulating object identifiers (IDs, filenames, tokens) in URLs or parameters.
  ::

  ::field{name="Directory Traversal" type="High"}
  Navigate outside web root using `../` sequences to access system files. Often combined with LFI.
  ::

  ::field{name="XML External Entity (XXE)" type="High"}
  Exploit XML parsers to read files, perform SSRF, or achieve denial of service via billion laughs attack.
  ::

  ::field{name="Server-Side Template Injection (SSTI)" type="Critical"}
  Inject template expressions (Jinja2, Twig, Freemarker) that execute on the server. Direct path to RCE.
  ::

  ::field{name="Insecure Deserialization" type="Critical"}
  Manipulate serialized objects (Java, PHP, Python pickle, .NET) to achieve RCE or privilege escalation.
  ::

  ::field{name="Authentication Bypass" type="Critical"}
  Default credentials, SQL injection in login forms, JWT manipulation, cookie tampering, session fixation.
  ::

  ::field{name="Information Disclosure" type="Medium"}
  Verbose error messages, stack traces, debug pages, exposed `.git`, `.env` files, server headers revealing versions.
  ::

  ::field{name="HTTP Request Smuggling" type="High"}
  Exploit discrepancies between front-end and back-end servers in parsing HTTP requests. Bypass security controls, poison caches, hijack requests.
  ::

  ::field{name="WebSocket Vulnerabilities" type="Medium"}
  CSWSH (Cross-Site WebSocket Hijacking), injection through WebSocket messages, missing authentication on WS endpoints.
  ::
::

## Defense & Remediation

::tip
Provide these **remediation recommendations** in your pentest report.
::

::card-group
  ::card
  ---
  icon: i-lucide-lock
  title: "Enforce HTTPS"
  color: green
  ---
  Redirect all HTTP (80) to **HTTPS (443)**. Implement HSTS headers. Never serve sensitive content over cleartext HTTP.
  ::

  ::card
  ---
  icon: i-lucide-shield-check
  title: "Security Headers"
  color: green
  ---
  Implement **CSP, X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, Referrer-Policy, Permissions-Policy**.
  ::

  ::card
  ---
  icon: i-lucide-scan
  title: "Input Validation"
  color: green
  ---
  **Validate and sanitize all user input** server-side. Use parameterized queries, output encoding, and allowlist validation.
  ::

  ::card
  ---
  icon: i-lucide-refresh-cw
  title: "Patch Management"
  color: green
  ---
  Keep web servers, frameworks, CMS, and plugins **up to date**. Subscribe to security advisories. Automate patching where possible.
  ::

  ::card
  ---
  icon: i-lucide-shield
  title: "WAF Deployment"
  color: green
  ---
  Deploy a **Web Application Firewall** (ModSecurity, Cloudflare, AWS WAF) to block common attack patterns.
  ::

  ::card
  ---
  icon: i-lucide-key-round
  title: "Authentication Hardening"
  color: green
  ---
  Implement **MFA**, strong password policies, account lockout, rate limiting, and secure session management.
  ::
::
