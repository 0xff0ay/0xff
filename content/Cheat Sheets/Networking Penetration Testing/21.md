## Overview

::card
---
icon: i-lucide-info
title: What is Port 21?
---
**FTP** (File Transfer Protocol) operates on **TCP port 21** (control channel) and **TCP port 20** (data channel in active mode) as one of the oldest network protocols still in widespread use. FTP is designed for file transfer between client and server — uploading, downloading, listing, and managing files remotely. It transmits **all data including credentials in cleartext**, making it inherently insecure. Despite being largely replaced by SFTP and SCP, FTP remains widespread on **web hosting servers, legacy enterprise systems, network equipment, printers, NAS devices, embedded systems, and IoT devices**. FTP is a **high-value target** — it often provides direct access to **web roots, configuration files, sensitive documents, and database backups**.
::

::callout
---
icon: i-lucide-skull
color: red
---
FTP sends **everything — usernames, passwords, commands, and file contents — in plaintext**. Any attacker on the same network can passively sniff credentials. Worse, **anonymous FTP access** is still frequently enabled, providing unauthenticated access to files. FTP with write access to a web root is a **direct path to remote code execution**.
::

### Nmap NSE Scripts

::note
Nmap ships with **multiple FTP-specific NSE scripts** covering anonymous access, brute force, bounce attacks, vulnerability detection, and more.
::

::code-group
```bash [All FTP Scripts]
# Run ALL FTP scripts
nmap -p 21 --script="ftp-*" <TARGET_IP>

# Safe enumeration combo
nmap -p 21 --script="ftp-anon,ftp-syst,ftp-bounce,\
ftp-libopie,ftp-proftpd-backdoor,ftp-vsftpd-backdoor" <TARGET_IP>
```

```bash [Anonymous Access]
# Check anonymous FTP login
nmap -p 21 --script=ftp-anon <TARGET_IP>

# Anonymous with directory listing
nmap -p 21 --script=ftp-anon \
  --script-args ftp-anon.maxlist=100 <TARGET_IP>
```

```bash [Brute Force]
# FTP brute force
nmap -p 21 --script=ftp-brute \
  --script-args userdb=users.txt,passdb=passwords.txt <TARGET_IP>

# With specific timeout
nmap -p 21 --script=ftp-brute \
  --script-args "userdb=users.txt,passdb=passwords.txt,\
ftp-brute.timeout=10s" <TARGET_IP>
```



## Weakness Analysis

### Core Vulnerabilities

::card-group
  ::card
  ---
  icon: i-lucide-eye-off
  title: "1. Cleartext Transmission"
  color: red
  ---
  **All FTP traffic — credentials, commands, and file data — is transmitted in plaintext.** Any MITM position allows complete credential theft and data interception. Even FTPS (explicit TLS) can be stripped.
  ::

  ::card
  ---
  icon: i-lucide-user-round-x
  title: "2. Anonymous Access"
  color: red
  ---
  **Anonymous FTP** is frequently enabled by default or misconfiguration. Provides unauthenticated file listing, download, and sometimes **upload/write access** to sensitive directories.
  ::

  ::card
  ---
  icon: i-lucide-upload
  title: "3. Write Access to Web Root"
  color: red
  ---
  FTP with write access to a web server's document root (`/var/www/html`, `htdocs`, `wwwroot`) allows **direct web shell upload** — the fastest path to RCE on web servers.
  ::

  ::card
  ---
  icon: i-lucide-key
  title: "4. Weak Authentication"
  color: red
  ---
  No built-in rate limiting, account lockout, or MFA. **Brute force attacks** are highly effective against FTP. Many implementations use system-level authentication (PAM), meaning FTP credentials = system credentials.
  ::

  ::card
  ---
  icon: i-lucide-bug
  title: "5. Legacy Software Vulnerabilities"
  color: red
  ---
  Many FTP servers (especially on embedded devices) run **ancient, unpatched versions** with known RCE vulnerabilities — vsftpd 2.3.4 backdoor, ProFTPD mod_copy, Wu-FTPd overflows.
  ::

  ::card
  ---
  icon: i-lucide-arrow-left-right
  title: "6. FTP Bounce Attack"
  color: orange
  ---
  FTP `PORT` command allows using the FTP server as a **proxy to scan internal networks** or **connect to other internal services** that are otherwise unreachable.
  ::
::

### Detailed Attack Surface

::field-group
  ::field{name="Credential Sniffing" type="Critical"}
  Passive capture of plaintext FTP credentials using `tcpdump`, `Wireshark`, `ettercap`, or `dsniff`. Zero skill required. Captures username, password, and all transferred files.
  ::

  ::field{name="Anonymous Access" type="Critical"}
  Unauthenticated read (and sometimes write) access. Can expose source code, configuration files, database backups, SSH keys, and more.
  ::

  ::field{name="Web Shell Upload" type="Critical"}
  FTP write access to web root = upload PHP/ASP/JSP shell → browse to shell → full RCE. The classic web server compromise chain.
  ::

  ::field{name="Brute Force" type="High"}
  No rate limiting on most FTP servers. Hydra/Medusa can test thousands of credentials per minute. System-level accounts often have weak passwords.
  ::

  ::field{name="FTP Bounce Scan" type="High"}
  Use `PORT` command to scan internal hosts through the FTP server. Bypass firewalls and map internal networks.
  ::

  ::field{name="FTP Bounce Attack (Data Exfil)" type="High"}
  Use FTP bounce to send data to arbitrary hosts/ports. Can exfiltrate data or attack internal services.
  ::

  ::field{name="Directory Traversal" type="High"}
  Some FTP servers allow escaping the FTP root via `../` sequences, accessing system files outside the intended directory.
  ::

  ::field{name="MITM / SSL Stripping" type="High"}
  Strip FTPS (explicit TLS) via MITM. Downgrade STARTTLS to cleartext FTP. Intercept all credentials and data.
  ::

  ::field{name="Passive Port Range Abuse" type="Medium"}
  Predictable or wide passive port ranges can be targeted. If the FTP server and web server share an IP, passive data channels may conflict with or expose other services.
  ::

  ::field{name="Symbolic Link Attack" type="Medium"}
  On misconfigured servers, symlinks within the FTP directory can point to sensitive system directories, exposing `/etc/`, `/root/`, etc.
  ::

  ::field{name="SITE EXEC Command" type="High"}
  Some FTP servers support `SITE EXEC` which allows direct command execution on the server through the FTP control channel.
  ::

  ::field{name="Denial of Service" type="Medium"}
  Connection exhaustion, resource-intensive operations (recursive listings), globbing attacks, oversized filenames, and FTP-specific protocol abuse.
  ::
::

## Defense & Remediation

::tip
Provide these **remediation recommendations** in your pentest report for FTP findings.
::

::card-group
  ::card
  ---
  icon: i-lucide-replace
  title: "Replace FTP with SFTP/SCP"
  color: green
  ---
  **Eliminate FTP entirely.** Replace with **SFTP (SSH File Transfer Protocol)** or **SCP** which provide encryption, strong authentication, and no separate data channel complications. This is the **#1 recommendation**.
  ::

  ::card
  ---
  icon: i-lucide-user-round-x
  title: "Disable Anonymous Access"
  color: green
  ---
  Disable anonymous FTP login. If required for a specific use case, restrict to **read-only access** on a dedicated, isolated directory with **no sensitive files**.
  ::

  ::card
  ---
  icon: i-lucide-lock
  title: "If FTP Required: Use FTPS"
  color: green
  ---
  If FTP cannot be replaced, enforce **FTPS (FTP over TLS)** with explicit TLS (STARTTLS) or implicit TLS (port 990). Use strong TLS 1.2+ configuration. Require encrypted data and control channels.
  ::

  ::card
  ---
  icon: i-lucide-key-round
  title: "Strong Authentication"
  color: green
  ---
  Enforce **strong, unique passwords**. Implement **fail2ban** for FTP brute force protection. Use **IP allowlisting** to restrict FTP access to known admin IPs only.
  ::
::

::card-group
  ::card
  ---
  icon: i-lucide-folder-lock
  title: "Chroot / Jail Users"
  color: green
  ---
  Configure **chroot jail** for all FTP users. Prevent directory traversal outside the intended FTP root. Users should **never** be able to access system directories.
  ::

  ::card
  ---
  icon: i-lucide-shield-check
  title: "Separate from Web Root"
  color: green
  ---
  **Never** map FTP directories to web server document roots. If file upload is needed for web content, use a staging directory with **manual review** before publication.
  ::

  ::card
  ---
  icon: i-lucide-network
  title: "Network Segmentation"
  color: green
  ---
  Restrict FTP to **specific network segments**. Block port 21 at perimeter firewalls. Use **VPN** for remote FTP access. Disable FTP bounce (PORT command to non-client IPs).
  ::

  ::card
  ---
  icon: i-lucide-refresh-cw
  title: "Patch & Update"
  color: green
  ---
  Keep FTP server software **updated** to the latest version. Subscribe to security advisories for vsFTPd, ProFTPD, Pure-FTPd, or your specific FTP server.
  ::

  ::card
  ---
  icon: i-lucide-eye
  title: "Monitoring & Logging"
  color: green
  ---
  Enable **verbose FTP logging** (xferlog, access logs). Monitor for brute force attempts, anonymous access, unusual file uploads, and SITE command usage. Forward logs to SIEM.
  ::

  ::card
  ---
  icon: i-lucide-file-lock
  title: "File Permissions"
  color: green
  ---
  Apply **strict file permissions**. FTP users should have minimal privileges. Disable **SITE EXEC**, **SITE CHMOD** (unless required). Prevent upload of executable files (`.php`, `.asp`, `.jsp`).
  ::
::

### Hardened vsFTPd Configuration Reference

::code-collapse
```bash [vsftpd.conf_hardened]
# ═══════════════════════════════════════════════════════
#  HARDENED vsftpd.conf — REFERENCE CONFIGURATION
# ═══════════════════════════════════════════════════════

# Disable anonymous access
anonymous_enable=NO

# Allow local users
local_enable=YES

# Enable write (if needed — disable if read-only)
write_enable=YES

# Chroot all local users (CRITICAL)
chroot_local_user=YES
allow_writeable_chroot=YES
# Or use chroot_list for specific users:
# chroot_list_enable=YES
# chroot_list_file=/etc/vsftpd.chroot_list

# Restrict users to their home directory
local_root=/home/$USER/ftp

# Umask (restrict file permissions)
local_umask=077
file_open_mode=0755

# Passive mode configuration (restrict port range)
pasv_enable=YES
pasv_min_port=40000
pasv_max_port=40100
pasv_address=<PUBLIC_IP>

# Disable PORT command to non-client (prevent bounce)
port_enable=NO
connect_from_port_20=NO

# SSL/TLS (mandatory FTPS)
ssl_enable=YES
force_local_data_ssl=YES
force_local_logins_ssl=YES
ssl_tlsv1=NO
ssl_sslv2=NO
ssl_sslv3=NO
ssl_tlsv1_1=NO
ssl_tlsv1_2=YES
# ssl_tlsv1_3=YES  # If supported
ssl_ciphers=HIGH:!aNULL:!MD5:!RC4
rsa_cert_file=/etc/ssl/certs/vsftpd.pem
rsa_private_key_file=/etc/ssl/private/vsftpd.key
require_ssl_reuse=NO

# Connection limits
max_clients=50
max_per_ip=3
max_login_fails=3
delay_failed_login=10
delay_successful_login=0

# Timeout settings
idle_session_timeout=300
data_connection_timeout=120
accept_timeout=60
connect_timeout=60

# Logging
xferlog_enable=YES
xferlog_std_format=YES
xferlog_file=/var/log/vsftpd.log
log_ftp_protocol=YES
syslog_enable=YES
dual_log_enable=YES
vsftpd_log_file=/var/log/vsftpd_detailed.log

# Security
hide_ids=YES
use_localtime=YES
seccomp_sandbox=YES
isolate_network=YES
isolate=YES

# Disable dangerous features
async_abor_enable=NO
ascii_upload_enable=NO
ascii_download_enable=NO
ls_recurse_enable=NO

# User control
userlist_enable=YES
userlist_deny=NO
userlist_file=/etc/vsftpd.userlist
# Only users in /etc/vsftpd.userlist can log in

# Banner (no version disclosure)
ftpd_banner=FTP Server Ready.
banner_file=/etc/vsftpd.banner

# PAM authentication
pam_service_name=vsftpd

# Listen
listen=YES
listen_ipv6=NO
listen_address=<BIND_IP>
listen_port=21

# Deny specific users
# Add root, bin, daemon, etc. to /etc/ftpusers
```
