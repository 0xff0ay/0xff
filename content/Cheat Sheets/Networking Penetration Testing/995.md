## Overview

POP3S (POP3 over SSL/TLS) is the encrypted variant of the Post Office Protocol version 3, running on **TCP port 995** with implicit TLS encryption. Unlike port 110 POP3 which optionally upgrades via STLS, port 995 establishes the TLS tunnel **before** any POP3 commands are exchanged. POP3 differs fundamentally from IMAP — it **downloads and typically deletes** messages from the server rather than managing them remotely. POP3S is a high-value penetration testing target because it grants access to **complete email contents**, often contains credentials in password reset emails, and the download-and-delete model means a single successful authentication can exfiltrate an entire mailbox before the legitimate user notices.

  ::card

## POP3 Fundamentals

### POP3 vs IMAP

::collapsible

| Feature | POP3/POP3S | IMAP/IMAPS |
| ------- | ---------- | ---------- |
| Default Port | 110 / 995 | 143 / 993 |
| Server Storage | Downloads & optionally deletes | Keeps on server |
| Folder Support | Single INBOX only | Multiple folders |
| Multi-Client | Poor (one client at a time) | Excellent |
| Offline Access | Designed for offline use | Designed for online use |
| Command Set | Simple (~12 commands) | Complex (~30+ commands) |
| Search | No server-side search | Full server-side search |
| Pentest Value | Quick full download, destructive | Persistent access, searchable |
| Common Use | Legacy systems, simple clients | Modern mail clients |

::

### POP3 Command Reference

::code-collapse

```text [pop3-commands.txt]
=== AUTHORIZATION STATE (Before Authentication) ===
CAPA                    List server capabilities
USER <username>         Set username for login
PASS <password>         Set password for login (after USER)
APOP <user> <digest>    APOP authentication (MD5)
AUTH <mechanism>        SASL authentication (PLAIN, LOGIN, NTLM, etc.)
STLS                    Upgrade to TLS (port 110 only, not on 995)
QUIT                    Close connection

=== TRANSACTION STATE (After Authentication) ===
STAT                    Get mailbox status (count + size)
LIST                    List all messages with sizes
LIST <msg#>             List specific message size
RETR <msg#>             Retrieve full message
DELE <msg#>             Mark message for deletion
NOOP                    No operation (keep-alive)
RSET                    Unmark all messages marked for deletion
TOP <msg#> <lines>      Retrieve headers + N lines of body
UIDL                    List unique IDs for all messages
UIDL <msg#>             Get unique ID for specific message
QUIT                    Close connection (commits deletions)

=== RESPONSE CODES ===
+OK                     Command succeeded
-ERR                    Command failed
+OK ... octets          Success with byte count

=== AUTHENTICATION EXAMPLES ===
USER admin              # Set username
PASS secret123          # Send password

AUTH PLAIN              # SASL PLAIN (base64 of \0user\0pass)
AUTH LOGIN              # SASL LOGIN (base64 user, then base64 pass)
AUTH NTLM               # NTLM authentication (Exchange)
AUTH XOAUTH2            # OAuth2 token authentication
APOP user <md5digest>   # APOP (MD5 of timestamp + password)
```

### SSL/TLS Certificate Analysis

::warning
POP3S certificate analysis is critical for reconnaissance. Certificates reveal **hostnames, internal domain names, organizational details, CA chains**, and Subject Alternative Names that expose infrastructure components invisible from the outside.
::

::tabs
  :::tabs-item{icon="i-lucide-shield" label="Certificate Extraction"}
  ```bash [Terminal]
  # Full certificate text dump
  openssl s_client -connect <target>:995 2>/dev/null | openssl x509 -noout -text

  # Subject and Issuer
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -subject -issuer

  # Subject Alternative Names (discover hostnames)
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -ext subjectAltName

  # All hostnames from certificate
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -subject -ext subjectAltName | \
    grep -oP '(?:DNS:|CN\s*=\s*)\K[^\s,]+' | sort -u

  # Validity dates
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -dates

  # Check if expired
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -checkend 0 && echo "[OK] Valid" || echo "[!!!] EXPIRED"

  # Check if self-signed
  SUBJ=$(openssl s_client -connect <target>:995 2>/dev/null | openssl x509 -noout -subject 2>/dev/null)
  ISS=$(openssl s_client -connect <target>:995 2>/dev/null | openssl x509 -noout -issuer 2>/dev/null)
  echo "$SUBJ" ; echo "$ISS"
  # If subject == issuer → self-signed

  # Serial number and fingerprint
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -noout -serial -fingerprint -sha256

  # Full certificate chain
  openssl s_client -connect <target>:995 -showcerts 2>/dev/null

  # Download certificate
  openssl s_client -connect <target>:995 2>/dev/null | \
    openssl x509 -outform PEM > pop3s-cert.pem

  # Nmap SSL certificate
  nmap -p 995 --script ssl-cert <target>

  # Certificate transparency log search
  curl -s "https://crt.sh/?q=%.target.com&output=json" | \
    jq -r '.[].name_value' | sort -u
  ```

  :::tabs-item{icon="i-lucide-info" label="NTLM Info Extraction"}
  ```bash [Terminal]
  # Extract domain info from NTLM auth on POP3S (Exchange)
  # No credentials needed

  nmap -p 995 --script pop3-ntlm-info <target>

  # Extracts:
  # - Target_Name (NetBIOS Domain)
  # - NetBIOS_Domain_Name
  # - NetBIOS_Computer_Name
  # - DNS_Domain_Name (FQDN)
  # - DNS_Computer_Name (Server FQDN)
  # - DNS_Tree_Name (Forest name)
  # - Product_Version (OS version)

  # Manual NTLM probe via Python
  python3 << 'PYEOF'
  import socket, ssl, base64, struct

  target = "<target>"
  port = 995

  ctx = ssl.create_default_context()
  ctx.check_hostname = False
  ctx.verify_mode = ssl.CERT_NONE

  sock = socket.create_connection((target, port), timeout=10)
  ssock = ctx.wrap_socket(sock, server_hostname=target)

  banner = ssock.recv(4096)
  print(f"[*] Banner: {banner.decode().strip()}")

  # Initiate NTLM auth
  ssock.send(b"AUTH NTLM\r\n")
  resp = ssock.recv(1024)
  print(f"[*] Auth prompt: {resp.decode().strip()}")

  # Send NTLM Type 1 (Negotiate)
  type1 = base64.b64encode(
      b"NTLMSSP\x00"
      b"\x01\x00\x00\x00"
      b"\x97\x82\x08\xe2"
      b"\x00\x00\x00\x00\x00\x00\x00\x00"
      b"\x00\x00\x00\x00\x00\x00\x00\x00"
  )
  ssock.send(type1 + b"\r\n")
  challenge_resp = ssock.recv(4096).decode().strip()
  print(f"[*] NTLM Type 2 received")

  # Parse Type 2 for domain info
  try:
      challenge_b64 = challenge_resp.replace("+ ", "").strip()
      challenge = base64.b64decode(challenge_b64)

      if challenge[0:7] == b"NTLMSSP":
          target_info_offset = struct.unpack("<I", challenge[40:44])[0]
          target_info_len = struct.unpack("<H", challenge[38:40])[0]
          target_info = challenge[target_info_offset:target_info_offset + target_info_len]

          pos = 0
          names = {1: "NetBIOS_Domain", 2: "NetBIOS_Computer",
                   3: "DNS_Domain", 4: "DNS_Computer", 5: "DNS_Tree"}
          while pos < len(target_info):
              av_id = struct.unpack("<H", target_info[pos:pos+2])[0]
              av_len = struct.unpack("<H", target_info[pos+2:pos+4])[0]
              av_value = target_info[pos+4:pos+4+av_len]
              if av_id in names:
                  print(f"    {names[av_id]}: {av_value.decode('utf-16-le')}")
              if av_id == 0:
                  break
              pos += 4 + av_len
  except Exception as e:
      print(f"[-] Parse error: {e}")

  ssock.send(b"*\r\n")
  ssock.send(b"QUIT\r\n")
  ssock.close()
  PYEOF
  ```

### APOP Authentication Attack

::note
APOP uses MD5(`<timestamp>` + `password`) for authentication. If you can capture the APOP challenge-response, the password can be cracked offline.
::

```bash [Terminal]
# Check if APOP is supported (banner contains timestamp)
echo "" | openssl s_client -connect <target>:995 -quiet 2>/dev/null | head -1
# +OK <12345.67890@mail.corp.local> = APOP supported

# APOP authentication format:
# APOP <username> <md5(timestamp+password)>

# If you capture an APOP exchange (via MITM on port 110):
# Server: +OK <1234.5678@mail.corp.local>
# Client: APOP user 5eb63bbbe01eeed093cb22bb8f5acdc3

# Crack the APOP hash:
# md5(timestamp + password) = captured_hash
```

## Pentest Checklist

::steps{level="4"}

#### Port Discovery and Service Identification

Scan for port 995 and related mail ports (110, 143, 993). Capture TLS banner, CAPA response, and identify the POP3 server software (Dovecot, Exchange, Cyrus, qpopper) from banner and capabilities.

#### SSL/TLS Certificate Analysis

Extract and analyze the certificate for hostnames (CN and SANs), issuer, validity dates, self-signed status, and organizational information. Certificate SANs often reveal internal infrastructure.

#### TLS Configuration Audit

Test for deprecated protocols (SSLv3, TLS 1.0, TLS 1.1), weak cipher suites, and known TLS vulnerabilities (Heartbleed, POODLE, CCS Injection, Logjam, FREAK, CRIME).

#### Enumerate POP3 Capabilities and Auth Mechanisms

Send CAPA command to identify supported authentication methods (USER/PASS, APOP, SASL PLAIN, LOGIN, CRAM-MD5, NTLM, GSSAPI, XOAUTH2) and protocol extensions (TOP, UIDL, PIPELINING).

#### Extract NTLM Information (Exchange)

Use `pop3-ntlm-info` Nmap script to obtain domain name, hostname, FQDN, forest name, and OS version from Exchange POP3S without authentication.

#### Check for APOP Support

Look for a timestamp in the POP3 banner (`<...>`). APOP support indicates MD5-based authentication that can be attacked via offline cracking if MITM is achieved.

#### Test Default and Common Credentials

Try default passwords for the identified server. Test `username`, `user@domain.com`, and `DOMAIN\user` formats.

#### Brute Force / Password Spray

Use Hydra, Metasploit, or custom scripts against port 995. POP3 lockout policies are often more lenient than AD, making brute force more effective.

#### Enumerate Users via SMTP

If SMTP runs on the same host, use VRFY/RCPT TO enumeration to build a user list for POP3S attacks.

#### Authenticate and Enumerate Mailbox

After successful login, use STAT, LIST, UIDL to assess mailbox contents. Use TOP to preview headers before full retrieval.

#### Exfiltrate Complete Mailbox

Download all emails using RETR commands. Use RSET before QUIT to preserve messages. Extract attachments for analysis.

#### Mine Email Content for Credentials

Search downloaded emails for plaintext passwords, password reset links, API keys, private keys, connection strings, internal IPs/hostnames, VPN configs, and financial data.

#### Test for Known Vulnerabilities

Check for CVEs specific to the identified server: Dovecot buffer overflow (CVE-2019-11500), Dovecot privilege escalation (CVE-2022-30550), Dovecot POP3 NULL deref (CVE-2020-10967), APOP weakness (CVE-2007-1558).

#### Test Credential Reuse

Try discovered credentials against IMAP, SMB, RDP, SSH, VPN, LDAP, web applications, and other services.

#### Document All Findings

Record evidence of weak TLS, self-signed/expired certificates, credential exposure, sensitive email content, and provide remediation recommendations.
