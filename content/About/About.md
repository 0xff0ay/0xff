## :icon{.text-primary name="i-lucide-swords"} What I [Do]{.text-primary}
::

::card-group
  :::card{color="red" icon="i-lucide-scan-eye" title="Zero Day Research"}
  Discovering previously unknown vulnerabilities in production software, operating systems, and network protocols through fuzzing, reverse engineering, and manual code auditing. Multiple **CVEs published** and coordinated with vendors worldwide.
  :::

  :::card
  ---
  color: primary
  icon: i-lucide-shield-alert
  title: Offensive Security Engineering
  ---
  Designing and executing advanced penetration tests, adversary simulations, and security assessments against web applications, APIs, mobile apps, networks, and cloud infrastructure for **Fortune 500 companies**.
  :::

  :::card{color="blue" icon="i-lucide-cloud" title="Cloud Security Engineering"}
  Securing and attacking **AWS, Azure, and GCP** environments. IAM policy auditing, container security, Kubernetes hardening, serverless exploitation, and cloud-native architecture review.
  :::

  :::card{color="orange" icon="i-lucide-target" title="Red Team Operations"}
  End-to-end adversary simulation — from initial access through phishing, lateral movement via **Active Directory abuse**, persistence mechanisms, data exfiltration, and C2 infrastructure management.
  :::

  :::card{color="purple" icon="i-lucide-binary" title="Reverse Engineering"}
  Static and dynamic analysis of binaries, malware samples, firmware, and proprietary protocols using **IDA Pro, Ghidra, x64dbg, Frida**, and custom tooling to uncover hidden vulnerabilities.
  :::

  :::card{color="red" icon="i-lucide-code" title="Exploit Development"}
  Crafting reliable exploits for stack overflows, heap corruption, use-after-free, type confusion, and logic bugs — bypassing modern mitigations like **ASLR, DEP, CFI, CET**, and kernel protections.
  :::

  :::card{color="neutral" icon="i-lucide-server" title="Systems Engineering"}
  Building and managing **high-availability infrastructure**, automating deployments, hardening Linux/Windows servers, configuring network security appliances, and maintaining production environments at scale.
  :::

  :::card{color="green" icon="i-lucide-sprout" title="Farming 🌾"}
  Yes, seriously. When I'm not in front of a terminal, I'm working the land. Farming keeps me **grounded** (literally), patient, and reminds me that not everything runs on electricity. 🌱
  :::
