#cloud-config

  -smp 2 \
  -hda ubuntu-vm.qcow2 \

  -enable-kvm \
  -nographic

## :icon{name="i-lucide-settings"} How to Create Your Own Mirror Site

::tip
Running a mirror benefits your organization with faster downloads, reduced external bandwidth, and offline access to OS repositories.
::

### Requirements

::field-group
  ::field{name="storage" type="string"}
  Minimum **2 TB** for a single distro. **20 TB+** for a multi-distro mirror. SSD recommended for metadata.
  ::

  ::field{name="bandwidth" type="string"}
  **1 Gbps** minimum. **10 Gbps** recommended for public-facing mirrors.
  ::

  ::field{name="server_os" type="string"}
  Debian or Ubuntu Server with `rsync`, `nginx`, and `cron` installed.
  ::

  ::field{name="sync_interval" type="string"}
  Every **4–6 hours** via rsync from official upstream mirrors.
  ::

  ::field{name="uptime" type="string"}
  **99.9%** uptime required for official mirror registration.
  ::
::

### Step-by-Step Setup

::steps{level="4"}

#### Install required packages

```bash [Terminal]
sudo apt update
sudo apt install -y rsync nginx certbot python3-certbot-nginx cron
sudo mkdir -p /srv/mirror/{ubuntu,debian,arch,kali,fedora}
sudo mkdir -p /var/log/mirror
```

#### Create sync scripts

```bash [/opt/mirror/sync-ubuntu.sh]
#!/bin/bash
LOCK="/tmp/ubuntu-sync.lock"
[ -f "$LOCK" ] && exit 1
touch "$LOCK" && trap "rm -f $LOCK" EXIT

rsync -avz --delete --timeout=600 \
  rsync://archive.ubuntu.com/ubuntu/ \
  /srv/mirror/ubuntu/ >> /var/log/mirror/ubuntu.log 2>&1
```

```bash [/opt/mirror/sync-kali.sh]
#!/bin/bash
LOCK="/tmp/kali-sync.lock"
[ -f "$LOCK" ] && exit 1
touch "$LOCK" && trap "rm -f $LOCK" EXIT

rsync -avz --delete --timeout=600 \
  rsync://archive.kali.org/kali/ \
  /srv/mirror/kali/ >> /var/log/mirror/kali.log 2>&1
```

```bash [/opt/mirror/sync-arch.sh]
#!/bin/bash
LOCK="/tmp/arch-sync.lock"
[ -f "$LOCK" ] && exit 1
touch "$LOCK" && trap "rm -f $LOCK" EXIT

rsync -avz --delete --timeout=600 \
  rsync://rsync.archlinux.org/ftp_tier1/ \
  /srv/mirror/arch/ >> /var/log/mirror/arch.log 2>&1
```

#### Configure nginx

```nginx [/etc/nginx/sites-available/mirror.conf]
server {
    listen 80;
    server_name mirror.example.com;
    root /srv/mirror;
    autoindex on;
    autoindex_exact_size off;
    autoindex_localtime on;

    location / {
        try_files $uri $uri/ =404;
    }

    access_log /var/log/nginx/mirror-access.log;
}
```

```bash [Terminal]
sudo ln -s /etc/nginx/sites-available/mirror.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d mirror.example.com
```

#### Schedule cron jobs

```bash [Terminal]
sudo crontab -e
```

```txt [crontab entries]
0 */6 * * * /opt/mirror/sync-ubuntu.sh
0 1,7,13,19 * * * /opt/mirror/sync-kali.sh
0 */4 * * * /opt/mirror/sync-arch.sh
0 6 * * * du -sh /srv/mirror/* >> /var/log/mirror/disk-usage.log
```

#### Run initial sync and register

```bash [Terminal]
# Make scripts executable
sudo chmod +x /opt/mirror/sync-*.sh

# Run first sync (may take hours)
sudo /opt/mirror/sync-ubuntu.sh &
sudo /opt/mirror/sync-kali.sh &

# Register your mirror:
# Ubuntu: https://launchpad.net/ubuntu/+newmirror
# Debian: https://www.debian.org/mirror/submit
# Arch:   https://wiki.archlinux.org/title/DeveloperWiki:NewMirrors
# Kali:   https://www.kali.org/docs/community/setting-up-a-kali-linux-mirror/
```

## :icon{name="i-lucide-shield-check"} Security Verification Reference

::card-group
  ::card
  ---
  title: SHA256 Checksum
  icon: i-lucide-hash
  ---
  `sha256sum -c SHA256SUMS --ignore-missing` — Verifies file integrity. Must match the official value.
  ::

  ::card
  ---
  title: GPG Signature
  icon: i-lucide-key
  ---
  `gpg --verify SHA256SUMS.gpg SHA256SUMS` — Confirms the checksum file was signed by the official project key.
  ::

  ::card
  ---
  title: HTTPS Only
  icon: i-lucide-lock
  ---
  Always download from HTTPS mirrors. HTTP connections can be intercepted and ISOs swapped by attackers.
  ::

  ::card
  ---
  title: Secure Boot
  icon: i-lucide-shield-alert
  ---
  Modern distros (Ubuntu, Fedora, Debian, openSUSE) ship signed UEFI bootloaders compatible with Secure Boot.
  ::
::
